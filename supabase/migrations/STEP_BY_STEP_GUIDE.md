# Step-by-Step Migration Guide

## 📚 الفرق بين Multi-Tenant و tenant_id

### Multi-Tenant Architecture (معمارية متعددة المستأجرين)
**المعنى:** نظام SaaS يسمح لعدة عملاء (tenants) باستخدام نفس التطبيق مع عزل كامل للبيانات.

**المكونات:**
- جدول `tenants` - يمثل كل عميل SaaS (مثل: شركة ABC، شركة XYZ)
- جدول `subscriptions` - الاشتراكات والخطط
- جدول `tenant_modules` - الموديولات المفعلة لكل عميل

**الاستخدام:** عندما تريد بيع النظام كخدمة SaaS لعدة عملاء.

---

### tenant_id (معرف المستأجر)
**المعنى:** حقل في كل جدول يربط السجل بـ tenant معين.

**الهدف:**
- عزل البيانات بين العملاء المختلفين
- السماح لعدة عملاء باستخدام نفس قاعدة البيانات
- تطبيق Row Level Security (RLS) بسهولة

**مثال:**
```sql
-- كل company مرتبطة بـ tenant_id
companies: id, name, tenant_id
-- كل customer مرتبط بـ tenant_id
customers: id, name, tenant_id
```

---

### الفرق العملي:

| Multi-Tenant | tenant_id |
|--------------|-----------|
| نظام كامل لإدارة العملاء | حقل في الجداول |
| جداول: tenants, subscriptions | حقل: tenant_id UUID |
| للأنظمة SaaS الكبيرة | للأنظمة متعددة العملاء |
| يحتاج إدارة اشتراكات | بسيط - فقط حقل إضافي |

**في حالتنا:** نستخدم **كليهما** - Multi-Tenant system + tenant_id في كل جدول
