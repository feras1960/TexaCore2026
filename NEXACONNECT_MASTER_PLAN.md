# 🔵 المرحلة الأولى — البنية الأساسية المتكاملة

> **الهدف:** نظام توكي ووكي كامل يعمل بشكل حقيقي مع مستخدمين حقيقيين

---

### 1.1 تزامن التسجيلات الصوتية ✅ مكتمل
**الجهد:** متوسط | **الأولوية:** الأعلى | **تم عبر LiveKit + Supabase Storage**

**الوضع الحالي:** التسجيلات تُحفظ محلياً فقط في المتصفح عبر `MediaRecorder` ← لا تظهر لباقي الأعضاء.

**المطلوب:**
- جدول `nexa_ptt_messages` (sender, audio_url, duration, channel_id, listened_by, expires_at)
- Supabase Storage bucket `ptt-recordings` للملفات الصوتية
- عند رفع اليد → رفع التسجيل → إدراج سجل → Realtime يُخبر الآخرين
- حذف تلقائي بعد 24 ساعة (Edge Function cron)
- الأعضاء يسمعون الرسائل الفائتة + علامة "مسموعة"

**الملفات:**
- `nexa_talkie_screen.dart` → `_stopTalking()` (رفع التسجيل)
- `ptt_service.dart` → `uploadVoiceMessage()`, `loadMessages()`
- Supabase: migration + storage bucket

---

### 1.2 ربط المجموعات والدعوات الحقيقية ✅ مكتمل
**الجهد:** متوسط | **الأولوية:** عالية | **تم ربطها بـ Supabase + LiveKit Rooms**

**الوضع الحالي:** `_mockChannels` ثابتة في الكود. `ptt_service.dart` جاهز لكن غير مربوط.

**المطلوب:**
- استبدال `_mockChannels` بـ `pttService.channels` من Supabase
- ربط `_joinConference()` بـ `channel.conference_room` الحقيقي
- ربط شاشة الدعوات بـ `pttService.pendingInvitations`
- ربط إنشاء مجموعة (`WizardStepMembers`) بـ `createChannel()` + `sendInvitation()`
- عند قبول الدعوة → يُضاف العضو → تظهر القناة في قائمته

---

### 1.3 قفل الميكروفون + مؤشر "فلان يتحدث" ✅ مكتمل
**الجهد:** بسيط | **الأولوية:** عالية | **يعمل عبر Supabase Realtime broadcast**

**الوضع الحالي:** كود Realtime جاهز في `ptt_service.dart` → `_subscribeToChannel()`. الواجهة فيها `_currentTalker` لكن غير مربوط.

**المطلوب:**
- عند `_startTalking()` → broadcast `ptt_talking: {user_name, is_talking: true}`
- الأعضاء الآخرون: يظهر اسم المتحدث + أنيميشن + يُقفل زر PTT
- عند `_stopTalking()` → broadcast `ptt_talking: {is_talking: false}` → يُفتح الزر
- سلوك اللاسلكي الكلاسيكي: شخص واحد فقط يتحدث في كل لحظة

### 1.4 حالات التواجد (Presence) 🟡 جزئي

**الوضع الحالي:** فلتر الحالات موجود في UI (متاح/تلقائي/صامت) + `nexa_ptt_members.is_online`

**المطلوب:**
- مزامنة حالة المستخدم مع Supabase عند تغييرها
- عرض عدد المتصلين الحقيقي لكل قناة
- أيقونات الحالة بجانب أسماء الأعضاء:
  - 🟢 متاح | 🔴 مشغول | 🟡 بعيد | ⚫ غير متصل
- عند `initState` → update `is_online: true`
- عند `dispose` → update `is_online: false`

# 🟣 المرحلة الثانية — الميزات المتقدمة والتنافسية

> **الهدف:** ميزات فريدة تجعل NexaTalkie أفضل من أي منافس بفضل تكامل ERP

---

### 2.1 زر الطوارئ (Emergency Alert) ⭐⭐⭐
**الجهد:** كبير | **الأولوية:** عالية

- ضغط مطول 3 ثواني → تأكيد → إنذار صوتي عالي لكل المجموعة
- إرسال GPS تلقائياً
- فتح قناة صوتية مباشرة
- رسالة تلقائية في الشات: "🚨 تنبيه طوارئ من أحمد"
- إشعار WhatsApp + SMS للمدير
- أنواع: SOS 🔴 | مساعدة 🟡 | عاجل 🔵
- سجل طوارئ مع الموقع والوقت
- **يحتاج:** جدول `nexa_ptt_emergencies` + Edge Function للـ SMS/WhatsApp + GPS API

---

### 2.2 إرسال صور + موقع أثناء PTT 📸📍
**الجهد:** متوسط | **الأولوية:** متوسطة

- أثناء الضغط على PTT: زر كاميرا → تلتقط صورة → ترسل فوراً
- زر موقع → GPS → خريطة تظهر للمجموعة
- مزيج Walkie-Talkie + WhatsApp
- **يحتاج:** Camera API + Geolocation API + Supabase Storage

---

### 2.3 Priority Speaker (أولوية التحدث) 👑
**الجهد:** متوسط | **الأولوية:** متوسطة

- مستويات: مدير (1) > مشرف (2) > موظف (3)
- عند ضغط المدير PTT → يُكتم الجميع تلقائياً
- مربوط بـ `nexa_ptt_members.role` (admin/member)
- **يحتاج:** تعديل Realtime logic + UI indicator

---

### 2.4 قنوات ذكية (Smart Channels) 🧠
**الجهد:** كبير | **الأولوية:** منخفضة

| النوع | الوصف |
|-------|-------|
| 📻 مفتوحة | الكل يتكلم ويسمع (الحالي) |
| 📢 استماع فقط | المدير يتكلم + الكل يسمع (PA System) |
| 🚨 طوارئ | مفتوحة دائماً، لا يمكن كتمها |
| ⏱️ مؤقتة | تنشأ لمهمة وتنتهي تلقائياً |
| 📍 جغرافية | تفعّل عند دخول منطقة (Geofencing) |

- **يحتاج:** حقل `channel_type` جديد + منطق Asterisk مختلف لكل نوع

---

### 2.5 Voice-to-Text (تحويل الصوت لنص) 🤖
**الجهد:** متوسط | **الأولوية:** منخفضة

- كل رسالة PTT تُحوّل تلقائياً لنص عربي/إنجليزي
- Whisper API (OpenAI) — $0.006/دقيقة
- البحث في النصوص أسرع من الصوت
- أرشفة نصية للمحادثات
- **يحتاج:** Edge Function + OpenAI API key

---

### 2.6 Lone Worker (العامل الوحيد) 🦺
**الجهد:** كبير | **الأولوية:** منخفضة (Enterprise)

- وضع خاص للعاملين وحدهم (حراسة ليلية، سائقين)
- فحص دوري: "هل أنت بخير؟" كل 30 دقيقة
- لم يرد → تنبيه للمدير + GPS + تفعيل ميكروفون
- مطلوب قانونياً في بعض الدول
- **يحتاج:** Background service + Push notifications + GPS tracking

---

### 2.7 دعم Bluetooth PTT 🎧
**الجهد:** متوسط | **الأولوية:** منخفضة (Native فقط)

- ربط زر PTT الفيزيائي في هواتف Samsung XCover, CAT, Blackview
- AirPods: ضغط مزدوج = PTT
- سماعات Bluetooth: اضغط واستمر = PTT
- **يحتاج:** Native Android/iOS فقط (لا يعمل على Web)

---

### 2.8 شبكات الجوال المخصصة (BLE Mesh Networking) 📡
**الجهد:** كبير | **الأولوية:** عالية (للطوارئ والمقاولات)

- **الفكرة:** استخدام تقنية Bluetooth Low Energy لنقل البيانات (P2P) في حال انقطاع الإنترنت.
- **التشغيل:** أي هاتف مثبت عليه تطبيق NexaConnect يعمل كـ "محطة تقوية" (Relay Node).
- **الاستخدامات:** إرسال نداء طوارئ (SOS)، رسائل نصية قصيرة، إحداثيات GPS، وحتى حزم صوتية صغيرة دون إنترنت!
- **يحتاج:** `nearby_connections` API أو `flutter_nearby_connections` (Native Android/iOS).

---

### 2.8 Dispatcher Console (لوحة المرسل) 🖥️
**الجهد:** كبير | **الأولوية:** متوسطة

- لوحة تحكم ويب للمدير في ERP
- رؤية كل القنوات النشطة + المتصلين + آخر النشاط
- التحدث لأي قناة من الكمبيوتر
- مراقبة مواقع الموظفين على خريطة
- تصدير التسجيلات
- **يحتاج:** مكون React جديد في ERP + ربط بـ PTT APIs

---

### 2.9 تكامل عميق مع ERP 🔗
**الجهد:** كبير | **الأولوية:** ميزة تنافسية فريدة

- "الشحنة وصلت" → زر تأكيد استلام → تحديث المخزون تلقائياً
- "كم سعر القماش الأزرق؟" → بحث في ERP → إرسال السعر
- GPS سائق → تحديث حالة التوصيل → إشعار للعميل
- **يحتاج:** API ربط بين PTT و ERP modules

---

### 2.10 Text Status (حالة نصية سريعة) 💬
**الجهد:** بسيط | **الأولوية:** منخفضة

- أزرار سريعة: [✅ تم] [🔄 جاري] [⏰ في الطريق] [🚫 مشكلة]
- ضغطة واحدة بدون كلام → تظهر للمجموعة
- **يحتاج:** Supabase Realtime broadcast + UI buttons
