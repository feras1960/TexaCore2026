### 1.1 تزامن التسجيلات الصوتية ✅ مكتمل
**الجهد:** متوسط | **الأولوية:** الأعلى | **تم عبر LiveKit + Supabase Storage**

**الوضع الحالي:** التسجيلات تُحفظ محلياً فقط في المتصفح عبر `MediaRecorder` ← لا تظهر لباقي الأعضاء.

**المطلوب:**
- جدول `nexa_ptt_messages` (sender, audio_url, duration, channel_id, listened_by, expires_at)
- Supabase Storage bucket `ptt-recordings` للملفات الصوتية
- عند رفع اليد → رفع التسجيل → إدراج سجل → Realtime يُخبر الآخرين
- حذف تلقائي بعد 24 ساعة (Edge Function cron)
- الأعضاء يسمعون الرسائل الفائتة + علامة "مسموعة"

**الملفات:**
- `nexa_talkie_screen.dart` → `_stopTalking()` (رفع التسجيل)
- `ptt_service.dart` → `uploadVoiceMessage()`, `loadMessages()`
- Supabase: migration + storage bucket

---

### 1.2 ربط المجموعات والدعوات الحقيقية ✅ مكتمل
**الجهد:** متوسط | **الأولوية:** عالية | **تم ربطها بـ Supabase + LiveKit Rooms**

**الوضع الحالي:** `_mockChannels` ثابتة في الكود. `ptt_service.dart` جاهز لكن غير مربوط.

**المطلوب:**
- استبدال `_mockChannels` بـ `pttService.channels` من Supabase
- ربط `_joinConference()` بـ `channel.conference_room` الحقيقي
- ربط شاشة الدعوات بـ `pttService.pendingInvitations`
- ربط إنشاء مجموعة (`WizardStepMembers`) بـ `createChannel()` + `sendInvitation()`
- عند قبول الدعوة → يُضاف العضو → تظهر القناة في قائمته

---

### 1.3 قفل الميكروفون + مؤشر "فلان يتحدث" ✅ مكتمل
**الجهد:** بسيط | **الأولوية:** عالية | **يعمل عبر Supabase Realtime broadcast**

**الوضع الحالي:** كود Realtime جاهز في `ptt_service.dart` → `_subscribeToChannel()`. الواجهة فيها `_currentTalker` لكن غير مربوط.

**المطلوب:**
- عند `_startTalking()` → broadcast `ptt_talking: {user_name, is_talking: true}`
- الأعضاء الآخرون: يظهر اسم المتحدث + أنيميشن + يُقفل زر PTT
- عند `_stopTalking()` → broadcast `ptt_talking: {is_talking: false}` → يُفتح الزر
- سلوك اللاسلكي الكلاسيكي: شخص واحد فقط يتحدث في كل لحظة

# 🔵 المرحلة G — LiveKit SIP Gateway (ربط المقاسم) 🌐

> **الهدف:** توحيد كل الاتصالات (داخلية + خارجية) عبر LiveKit، مع إبقاء Asterisk كبوابة للخطوط الأرضية و GOIP

### المعمارية:
```
📱 التطبيق (NexaConnect)
  └── يتصل فقط بـ LiveKit (WebRTC)
       └── LiveKit SIP Service (على السيرفر)
            └── SIP Trunk ↔ Asterisk PBX
                 ├── خطوط أرضية (FXO)
                 ├── خطوط جوال (GOIP/SIM)
                 ├── IVR + طوابير انتظار
                 └── تحويلات + مؤتمرات
```

### الفوائد:
- **أداء خارق:** التطبيق لا يحتاج محرك SIP — خفيف على البطارية
- **جودة HD:** تحويل الصوت يتم على السيرفر القوي
- **مكالمات جماعية مدمجة:** إضافة زميل لمكالمة عميل خارجي بضغطة
- **تسجيل مركزي:** كل المكالمات (داخلية + خارجية) من نقطة واحدة
