| أمين المستودع | 📦 استلام + تسليم + جرد فقط — **لا يُنشئ طلبات مشتريات** |
| الدفع الجزئي | ✅ يُغيّر المرحلة تلقائياً: `posted → partial_paid → paid` |

## 📦 نمط الاستلام: محلي vs دولي

### المحلي (افتراضي) `receipt_mode = 'direct'`
```
1. تأكيد فاتورة أو أمر شراء
2. ⚡ إنشاء تلقائي لمستند استلام (stage = pending_receipt)
3. 🔔 إشعار لأمين المستودع:
   - فوراً: إذا لم يُحدد تاريخ استلام
   - في التاريخ: إذا حُدد expected_receipt_date
4. يظهر في تبويب "استلام" → حالة "بانتظار الاستلام"
5. أمين المستودع يؤكد → يُحدّث المخزون
```

### الدولي `receipt_mode = 'international'`
```
1. تأكيد أمر شراء/فاتورة + receipt_mode = 'international'
2. ربط بشحنة (shipment) → حاوية (container)
3. مستند الاستلام يُنشأ عند وصول الشحنة
4. أمين المستودع يؤكد الاستلام
```

### الاستلام الجزئي:
```
purchase_transaction_items:
  ordered_qty    ← الكمية المطلوبة (الأصلية)
  received_qty   ← الكمية المُستلمة حتى الآن
  remaining_qty  ← تُحسب (ordered_qty - received_qty)

عند الاستلام الجزئي:
  → يُحدّث received_qty لكل بند
  → يبقى receipt_status = 'partial'
  → عند استلام الكل: receipt_status = 'received'
```

### آلية المصاريف المرتبطة:

```typescript
// المصاريف تُجلب من سندات الصرف المرتبطة بالفاتورة
// لا تُدخل يدوياً هنا — تُدفع من الصندوق/البنك
const { data: linkedExpenses } = useQuery({
  queryKey: ['linked-expenses', transactionId],
  queryFn: () => supabase
    .from('treasury_transactions')  // أو payment_vouchers
    .select('*')
    .eq('linked_transaction_id', transactionId)
    .eq('transaction_type', 'expense')
    .order('date', { ascending: false }),

```

## 🎛️ الأزرار الوظيفية لكل مرحلة

### `draft` — مسودة
```
[▶ تأكيد كطلب مشتريات] [▶ تأكيد كعرض سعر]
[▶ تأكيد كأمر شراء]   [▶ تأكيد كفاتورة]
[🗑 حذف المسودة]
مؤشر: ● جاري الحفظ... / ✓ محفوظ
```

### `request` — طلب مشتريات
```
[✎ تعديل] [📄 تحويل لعرض سعر▼] [🛒 تحويل لأمر شراء▼]
[📄 تحويل لفاتورة▼] [🖨 طباعة] [🗑 حذف]
▼ = يطلب اختيار مورد
```

### `quotation` — عرض سعر
```
[✎ تعديل] [🛒 تحويل لأمر شراء] [📄 تحويل لفاتورة]
[📧 إرسال للمورد] [✕ رفض العرض] [🖨 طباعة]
```

### `order` — أمر شراء (قبل الاعتماد)
```
[✎ تعديل] [✓ اعتماد] [📧 إرسال للمورد] [🖨 طباعة]
[✕ إلغاء الأمر]
```

### `approved` — معتمد (بعد الاعتماد)
```
[📄 إنشاء فاتورة] [📦 تسجيل استلام]
[🖨 طباعة] [✕ إلغاء الاعتماد] (مدير فقط)
```

### 🔄 الدفع الجزئي — تحويل المرحلة تلقائياً:

```
      stage = 'partial_paid'  (تلقائياً)
  → إذا المبلغ المدفوع >= إجمالي الفاتورة:
      stage = 'paid'  (تلقائياً)

الكود:
  async function recordPayment(transactionId, amount) {
    // 1. تسجيل الدفعة في جدول المدفوعات
    // 2. حساب إجمالي المدفوع
    const totalPaid = previousPaid + amount;
    const grandTotal = transaction.grand_total;
    
    // 3. تحديث المرحلة تلقائياً
    const newStage = totalPaid >= grandTotal ? 'paid' : 'partial_paid';
    await updateStage(transactionId, newStage);
    
    // 4. تسجيل في النشاط
    await logActivity(transactionId, 'payment', { amount, totalPaid, newStage });
  }
```
