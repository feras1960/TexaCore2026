### 1️⃣ الإضافة المباشرة + بوب أب الكمية والسعر

#### الفكرة:
عند النقر على مادة في المتصفح → يفتح **Pop-up صغير** يحتوي:

```
┌─────────────────────────────────────────┐
│  📦 قطن سادة 100% (CST-100-PLATN)      │
│  ─────────────────────────────────────  │
│                                         │
│  🎨 اللون:   [أبيض ▼]                  │
│                                         │
│  📊 الرصيد الحالي:                      │
│     • أمتار: 260 متر                    │
│     • رولونات: 7                        │
│                                         │
│  💰 آخر أسعار الشراء:                   │
│  ┌──────────────────────────────────┐   │
│  │ المورد          │ السعر  │ التاريخ │   │
│  │─────────────────┼────────┼────────│   │
│  │ ★ تركي تكستيل  │ $45.00 │ 02/01  │   │
│  │   هندي فابريكس │ $42.50 │ 01/20  │   │
│  │   صيني سيلك    │ $40.00 │ 12/15  │   │
│  └──────────────────────────────────┘   │
│  ★ = آخر سعر (مُثبت افتراضياً)         │
│                                         │
│  🔢 الكمية:    [___10___] متر           │
│  💵 السعر:     [__45.00__] USD          │
│  💰 الإجمالي:  $450.00                  │
│                                         │
│      [إلغاء]          [✅ إضافة]        │
└─────────────────────────────────────────┘
```

#### التدفق:
```
1. المستخدم يتصفح المواد
2. ينقر على مادة → يفتح Pop-up
3. يختار اللون (إن وجد)
4. يرى آخر أسعار الموردين (مُرتبة بالأحدث)
5. آخر سعر يُثبت تلقائياً في حقل السعر
6. يُدخل الكمية
7. يضغط "إضافة" → تُضاف للفاتورة
8. Pop-up يُغلق → يعود لتصفح المواد
9. يمكن تكرار العملية لمواد أخرى
```

#### مصدر البيانات لآخر الأسعار:
```sql
-- آخر أسعار الشراء لمادة معينة
SELECT DISTINCT ON (s.name)
    s.name AS supplier_name,
    ti.unit_price,
    ti.currency,
    td.document_date,
    td.document_number
FROM trade_document_items ti
JOIN trade_documents td ON td.id = ti.document_id
JOIN suppliers s ON s.id = td.supplier_id
WHERE ti.material_id = :material_id
  AND td.document_type IN ('purchase_invoice', 'purchase_order')
  AND td.company_id = :company_id
ORDER BY s.name, td.document_date DESC;
```

### 2️⃣ تجميع المواد حسب الألوان (بدلاً من المستودعات)

#### الهيكل الحالي:
```
المادة: قطن سادة 100%
├── 📦 مستودع أوديسا: 168 متر | 5 رولونات
├── 📦 مستودع كييف: 100 متر | 2 رولونات
└── الإجمالي: 260 متر | 7 رولونات
```

#### الهيكل المطلوب:
```
المادة: قطن سادة 100%
├── 🎨 أبيض:  120 متر | 3 رولونات
├── 🎨 أحمر:   80 متر | 2 رولونات
├── 🎨 أزرق:   60 متر | 2 رولونات
└── الإجمالي: 260 متر | 7 رولونات
```

#### مصدر بيانات الألوان:
```
fabric_rolls table:
├── material_id → ربط بالمادة
├── color → اسم اللون
├── length_meters → الطول بالأمتار
├── status → (available, reserved, sold)
└── warehouse_id → المستودع (للعرض الداخلي فقط)

الاستعلام:
SELECT 
    color,
    COUNT(*) AS roll_count,
    SUM(length_meters) AS total_meters
FROM fabric_rolls
WHERE material_id = :id AND status = 'available'
GROUP BY color
ORDER BY total_meters DESC;
```

#### ملاحظة مهمة:
- **الأقمشة**: تعرض ألوان + أمتار + رولونات
- **المواد الأخرى** (كيماويات، إكسسوارات...): تعرض الرصيد العام فقط بدون تفصيل ألوان
- التحديد: `materials.category` أو `material_groups.type` يحدد هل المادة قماش أم لا

### 3️⃣ العملة الديناميكية

#### المشكلة الحالية:
- العملة تظهر SAR (ثابتة/hardcoded)
- لا تتغير حسب المادة أو المورد

#### الحل المطلوب — ترتيب الأولوية:

```
عملة الفاتورة = أولوية ↓
1. عملة المورد المختار (supplier.default_currency)
2. عملة المادة (materials.purchase_currency أو materials.currency)  
3. العملة الأساسية للشركة (company_settings.base_currency)
```

#### التدفق:
```
1. المستخدم يفتح فاتورة شراء جديدة
2. يختار المورد → العملة تتغير تلقائياً لعملة المورد
3. يمكنه تغيير العملة يدوياً
4. سعر الصرف يُحسب تلقائياً من جدول exchange_rates
5. عند تأكيد الفاتورة → يُثبت سعر الصرف
```

#### تأثير تغيير العملة:
```
إذا عملة الفاتورة ≠ العملة الأساسية للشركة:
├── يظهر حقل "سعر الصرف" (قابل للتعديل)
├── كل الأسعار تُعرض بعملة الفاتورة
├── الإجمالي يُعرض بالعملتين (عملة الفاتورة + العملة الأساسية)
└── القيد المحاسبي يُسجل بالعملة الأساسية
```

### 4️⃣ ربط بالكونتينر

#### التدفق:
```
فاتورة شراء → تأكيد → خيارات:
├── ✅ إنشاء قيد محاسبي تلقائي (حساب المورد)
├── 📦 ربط بكونتينر موجود
│   └── [اختر الكونتينر ▼] → يُضاف كـ container_items
└── 📦 إنشاء كونتينر جديد
    └── يفتح نموذج إنشاء كونتينر مع ربط الفاتورة
```

#### الجداول المعنية:
```
trade_documents (الفاتورة)
  ↓ container_id (حقل جديد أو موجود)
containers
  ↓
container_items (بنود الكونتينر من الفاتورة)
```

### 5️⃣ القيد المحاسبي التلقائي عند التأكيد

#### القيد:
```
عند تأكيد فاتورة الشراء:
┌────────────────────────────┬──────────┬──────────┐
│ الحساب                     │ مدين     │ دائن     │
│────────────────────────────┼──────────┼──────────│

- المبلغ بالعملة الأساسية: ₴185,400
- القيد يُسجل بـ ₴185,400
```

### 6️⃣ طلبات مشتريات الفروع (Branch Purchase Requests)

#### الفكرة الأساسية:
كل مدير فرع (كييف، خاركوف، أوديسا) يمكنه إنشاء طلب مشتريات من فرعه.
المدير العام يرى كل طلبات الفروع ويمكنه تجميعها في فاتورة/أمر شراء واحد.

│         ↓                      ↓                    ↓       │

│  ════════════════════════════════════════════════════════    │

### 8️⃣ نظام الحجز لحساب الفروع (Branch Reservation)

#### الفرق بين حجز الفرع وحجز الزبون:

```
نوعان من الحجز:
│
├── 🏪 حجز فرع (Branch Reservation) ← جديد
│   ├── المصدر: طلب مشتريات فرع
│   ├── الهدف: ضمان وصول البضاعة للفرع الصحيح
│   ├── الحالة: محجوز_لفرع → مستلم_بالفرع
│   ├── يتحول لمخزون الفرع عند الاستلام
│   └── لا يؤثر على حجوزات الزبائن
│
└── 👤 حجز زبون (Customer Reservation) ← موجود
    ├── المصدر: فاتورة مبيعات / أمر بيع
    ├── الهدف: ضمان توفر البضاعة للزبون
    ├── الحالة: محجوز → مُسلّم
    └── يبقى كما هو — لا تغيير
```

#### تدفق الحجز للفرع:

```
1. المدير يُضيف طلب فرع كييف للفاتورة
2. عند تأكيد الفاتورة:
   ├── حالة الطلب → "محجوز لحساب فرع كييف"
   ├── يُنشأ سجل في branch_reservations:
   │   { branch_id, material_id, color, quantity, source_document_id }
   └── مدير فرع كييف يرى: "طلبك محجوز ✅ — بانتظار الاستلام"
3. عند استلام البضاعة:
   ├── حالة الطلب → "مستلم"
   ├── المخزون يُضاف لمستودع كييف تلقائياً
   └── إشعار لمدير فرع كييف: "بضاعتك وصلت! 📦"
```

### 9️⃣ طلبات الزبائن من الفروع (Customer Orders from Branches)

#### الفكرة:
نفس تبويب طلبات الفروع يمكن استخدامه عند الفروع لتسجيل طلبات الزبائن.

#### واجهة طلب الزبون (عند الفرع):

```
┌─────────────────────────────────────────────────────┐
│ 🛒 طلب زبون — فرع كييف                             │
│─────────────────────────────────────────────────────│
│                                                     │
│ 👤 الزبون:    [____أحمد محمود____]                  │
│ 📱 الهاتف:   [____+380...._______]                 │
│ 🏪 الفرع:    فرع كييف (تلقائي)                     │
│ 📅 التاريخ:  12/02/2026                            │
│ 📋 الأولوية: [عادي ▼]                              │
│                                                     │
│ ═══ المواد المطلوبة ═══                             │
│ ┌────────┬───────┬──────┬──────┬─────────────────┐  │
│ │ المادة │ اللون │الكمية│الوحدة│ ملاحظات         │  │
│ │────────┼───────┼──────┼──────┼─────────────────│  │
│ │ قطن100%│ أبيض  │ 30   │ متر  │ للخياطة        │  │
│ │ حرير   │ ذهبي  │ 10   │ متر  │ فستان عرس      │  │
│ └────────┴───────┴──────┴──────┴─────────────────┘  │
│                                                     │
│ 💰 السعر التقديري: $1,250 (اختياري)                │
│                                                     │
│             [حفظ كمسودة]  [إرسال للمدير 📤]        │
└─────────────────────────────────────────────────────┘
```

#### العلاقة: طلب الزبون → طلب الشراء → الفاتورة

```
🛒 طلب زبون (Customer Request)
   ↓ يتحول إلى
📝 طلب شراء فرع (Branch Purchase Request) — إذا المادة غير متوفرة
   ↓ يظهر في
⭐ تبويب طلبات الفروع (عند المدير)
   ↓ يُضاف إلى
📄 فاتورة شراء مُجمّعة
   ↓ عند التأكيد
🔒 محجوز لحساب الفرع + لحساب الزبون
   ↓ عند الاستلام
📦 يدخل مستودع الفرع → إشعار للفرع → إشعار للزبون
```

## 📊 الهيكل الكامل لقاعدة البيانات (إضافات)

### جدول جديد: `branch_requests`

```sql
CREATE TABLE branch_requests (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    company_id UUID NOT NULL,
    branch_id UUID REFERENCES branches(id),        -- الفرع الطالب
    warehouse_id UUID REFERENCES warehouses(id),    -- المستودع المطلوب التوريد له
    
    -- نوع الطلب
    request_type TEXT NOT NULL DEFAULT 'purchase',  -- 'purchase' | 'customer_order'
    
    -- معلومات الزبون (لطلبات الزبائن)
    customer_name TEXT,
    customer_phone TEXT,
    customer_id UUID REFERENCES customers(id),      -- اختياري
    
    -- الحالة
    status TEXT DEFAULT 'draft',
    -- draft → submitted → approved → reserved → fulfilled → received | rejected
    
    priority TEXT DEFAULT 'normal',                 -- 'urgent' | 'normal' | 'planned'
    
    -- ربط بالمستند
    source_document_id UUID,                        -- فاتورة الشراء المُجمّعة
    source_document_type TEXT,                      -- 'purchase_invoice' | 'purchase_order'
    
    -- التواريخ
    requested_date TIMESTAMPTZ DEFAULT now(),
    needed_by_date DATE,                            -- تاريخ الحاجة
    approved_at TIMESTAMPTZ,
    fulfilled_at TIMESTAMPTZ,
    received_at TIMESTAMPTZ,
    
    -- المستخدمين
    requested_by UUID,                              -- مدير الفرع
    approved_by UUID,                               -- المدير العام
    
    notes TEXT,
    created_at TIMESTAMPTZ DEFAULT now(),
    updated_at TIMESTAMPTZ DEFAULT now()
);

-- بنود الطلب
CREATE TABLE branch_request_items (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    request_id UUID REFERENCES branch_requests(id) ON DELETE CASCADE,
    material_id UUID REFERENCES materials(id),
    color TEXT,                                      -- اللون المطلوب
    quantity NUMERIC NOT NULL,
    unit TEXT DEFAULT 'meter',                       -- meter | roll | piece | kg
    
    -- الحالة لكل بند
    status TEXT DEFAULT 'pending',
    -- pending → reserved → fulfilled → received
    
    -- ربط بالفاتورة
    invoice_item_id UUID,                            -- بند الفاتورة المرتبط
    reserved_quantity NUMERIC DEFAULT 0,
    received_quantity NUMERIC DEFAULT 0,
    
    -- ملاحظات
    customer_note TEXT,                              -- "لزبون أحمد" أو "فستان عرس"
    notes TEXT,
    
    created_at TIMESTAMPTZ DEFAULT now()
);
```

## 🔍 أسئلة للتوضيح قبل البدء

### متصفح المواد:
1. **آخر الأسعار**: كم سعر نعرض (آخر 3؟ آخر 5؟)
2. **الألوان**: الأقمشة فقط لها ألوان في `fabric_rolls`، صح؟
3. **المستودع في متصفح المواد**: يبقى كفلتر جانبي اختياري؟

### طلبات الفروع:
4. **الفروع**: كم فرع موجود حالياً وما أسماؤهم؟
5. **الصلاحيات**: مدير الفرع يرى فقط طلبات فرعه، والمدير العام يرى الكل — صح؟
6. **التجميع**: المدير يمكنه تعديل الكميات عند التجميع (مثلاً يزيد 10% احتياط)؟
7. **طلبات الزبائن**: هل يمكن للفرع إنشاء طلب بدون ربطه بزبون (طلب عام)؟

### العملة والقيد:
8. **القيد التلقائي**: يُنشأ عند "التأكيد" أم عند "الترحيل"؟
9. **الكونتينر**: ربط الفاتورة كاملة بكونتينر واحد، أم بنود محددة بكونتينرات مختلفة؟
