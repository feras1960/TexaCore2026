**التاريخ:** 2026-02-17 (تحديث 3)  
**الحالة:** Phase 1→4 مكتملة ✅ — Phase 5 التالية  
**الهدف:** ربط الفواتير بالكونتينر + قفل الفواتير + توزيع المصاريف + تحديث Landed Cost

---

## 📐 نظرة عامة على البنية

```
كونتينر واحد ← عدة فواتير مرحّلة ← أصناف تُنسخ لـ shipment_items
              ← عدة مصاريف (shipment_costs) ← توزيع على الأصناف
                                               ← تحديث تكلفة المخزون
```

### المبدأ الأساسي:
> **الفاتورة تُنهي عملها (ترحيل) ← تُربط بالكونتينر ← تُقفل ← البضائع بعهدة الكونتينر**

### الجداول الموجودة ✅:
| الجدول | الوظيفة |
|--------|---------|
| `shipments` | بيانات الكونتينر + status + cost_allocation |
| `shipment_items` | أصناف الكونتينر (مع purchase_invoice_id) |
| `shipment_costs` | مصاريف الكونتينر (expected vs actual) |
| `transit_reservations` | حجوزات العملاء على بضائع بالطريق |
| `purchase_transactions` | container_id + container_number + container_status |

### 📋 قواعد العمل الأساسية:

#### شروط الربط:
| الشرط | السبب |
|-------|-------|
| الفاتورة بحالة `posted` أو `partial_paid` أو `paid` | لضمان اكتمال البيانات |
| الفاتورة **غير مربوطة** بكونتينر آخر | منع التكرار |
| الكونتينر ليس بحالة `received` أو `closed` | لا إضافة لكونتينر مُغلق |

#### ماذا يحدث بعد الربط:
| الإجراء | متاح؟ |
|---------|-------|
| تعديل أصناف/أسعار الفاتورة | ❌ مقفلة |
| حذف الفاتورة | ❌ مقفلة |
| إلغاء الترحيل | ❌ مقفلة |
| عرض الفاتورة (read-only) | ✅ |
| فك الارتباط (بتأكيد) | ✅ → يُعيد الفاتورة لـ `posted` |
