# تحليل شامل: هيكل المستخدمين والصلاحيات

## 📊 الوضع الحالي — 7 تبويبات

| # | التبويب | الوظيفة الحالية | الحالة |
|---|---------|----------------|--------|
| 1 | **الأدوار** | إنشاء/تعديل أدوار + تحديد الأقسام المسموحة + صلاحيات CRUD للوحدات | ✅ يعمل |
| 2 | **المستخدمين** | قائمة المستخدمين + تعيين أدوار + ربط فروع/مستودعات/صناديق | ✅ يعمل |
| 3 | **الفروع** | إدارة الفروع (إضافة/تعديل/حذف) | ✅ يعمل |
| 4 | **صلاحيات خاصة** | مصفوفة صلاحيات خاصة لكل دور (حذف مرحّل، تعديل فترات...) | ✅ يعمل |
| 5 | **الموارد** | ربط مستخدم بصناديق/مستودعات/فروع محددة | ⚠️ تكرار مع تبويب المستخدمين |
| 6 | **قواعد الإخفاء** | إخفاء حقول/صفحات/أقسام عن أدوار معينة | ⚡ متقدم |
| 7 | **سجل التدقيق** | سجل تغييرات الصلاحيات | 🔧 placeholder فقط |

---

## 🔍 تحليل المشاكل

### 1. تكرار بين "المستخدمين" و "الموارد"
- **تبويب المستخدمين** (`UsersManagementTab`) يسمح بتعيين فروع/مستودعات/صناديق لكل مستخدم
- **تبويب الموارد** (`UserPermissionsTab`) يفعل **نفس الشيء بالضبط** + يعرض الأدوار
- النتيجة: **تكرار واضح** يربك المستخدم

### 2. "صلاحيات خاصة" يمكن دمجها مع "الأدوار"
- تبويب "الأدوار" يحتوي أصلاً على إعدادات للصلاحيات الخاصة (SPECIAL_PERMS_CONFIG)
- تبويب "صلاحيات خاصة" يعرض نفس البيانات بشكل matrix
- الأفضل: **المصفوفة تكون جزء من تبويب الأدوار** أو يبقى كتبويب مستقل للمعاينة السريعة

### 3. "قواعد الإخفاء" متقدمة جداً
- هذه ميزة Enterprise متقدمة
- تسمح بإخفاء حقول/صفحات/أقسام كاملة عن أدوار معينة
- مفيدة لكن ليست أولوية الآن

### 4. "سجل التدقيق" placeholder
- البيانات وهمية (hardcoded)
- غير مربوط بقاعدة البيانات

## ✅ الهيكل المقترح — 4 تبويبات أساسية

بما أننا **نعمل على مستوى الشركة**، الهيكل المثالي:

### 📋 التبويب 1: مجموعات المستخدمين (الأدوار)
> **الاسم**: "مجموعات المستخدمين" بدلاً من "الأدوار"

**الوظيفة**:
- قائمة بالأدوار المتاحة (مالك الشركة، مدير الشركة، مدير الفرع، محاسب، أمين صندوق...)
- لكل دور: الأقسام المسموحة + صلاحيات CRUD + الصلاحيات الخاصة
- **مصفوفة الصلاحيات الخاصة** مدمجة هنا (يمكن أن تبقى كجزء أو كـ sub-tab)

**المكونات الحالية**: `RolesManagementTab` + `SpecialPermissionsTab` (مدمج)

---

### 👥 التبويب 2: المستخدمين
> **الاسم**: "المستخدمين"

**الوظيفة**:
- قائمة المستخدمين في الشركة
- تعيين دور (مجموعة) لكل مستخدم
- تعيين فرع رئيسي + فروع إضافية
- تعيين مستودعات وصناديق (الموارد المسموحة)
- **تعديل صلاحيات فردية**: يمكن إضافة/إزالة صلاحيات خاصة للمستخدم تتجاوز صلاحيات الدور
- حالة المستخدم (فعال/معطل)

**المكونات الحالية**: `UsersManagementTab` + `UserPermissionsTab` (مدمج)

---

### 🏢 التبويب 3: الفروع
> **الاسم**: "الفروع"

**الوظيفة**:
- إدارة الفروع (إضافة/تعديل/حذف)
- لكل فرع: المستودعات التابعة + الصناديق + المستخدمين
- **عرض شجري**: فرع → مستودعاته → صناديقه → مستخدميه
- صلاحيات المستخدمين ضمن الفرع

**المكونات الحالية**: `BranchesManagementTab`

---

### 📋 التبويب 4: سجل التدقيق
> **الاسم**: "سجل التدقيق"

**الوظيفة**:
- سجل كامل لكل تغيير في الصلاحيات
- من غيّر ماذا ومتى
- مربوط بقاعدة البيانات

|--------|--------|
