### 1.1 حسابات بضاعة بالطريق (أصول — 1143)

```
1140 المخزون (مجموعة) [موجود]
├── 1141 بضاعة جاهزة [موجود]
├── 1142 مواد خام [موجود]

    ├── 11432 كونتينر TEXU-2025-002  ← يُنشأ تلقائياً بـ Trigger
    └── ...

### 1.2 حسابات مصاريف الكونتينرات (مصروفات — 58xx)

```
5000 المصروفات (مجموعة) [موجود]
├── ...
├── 5800 مصاريف الشحنات والاستيراد (مجموعة — جديد)
│   ├── 5810 شحن بحري / Ocean Freight
│   ├── 5820 رسوم جمارك / Customs Duties
│   ├── 5830 تأمين بحري / Marine Insurance
│   ├── 5840 نقل داخلي / Inland Transport
│   ├── 5850 تخليص جمركي / Customs Clearance
│   ├── 5860 فحص وتفتيش / Inspection & Testing
│   ├── 5870 رسوم ميناء / Port Charges
│   ├── 5880 تخزين / Storage & Warehousing
│   └── 5890 مصاريف شحنات أخرى / Other Shipping Expenses
└── 5900 مصاريف أخرى [موجود — يُعاد ترقيمه إلى 5950 في الشجرة الجديدة فقط]

### 1.3 ملاحظة: تعارض رقم 5800

> **الشجرة البسيطة الحالية**: `5800` = مصاريف مالية (Financial Expenses)
> **الحل**: في الشجرات الجديدة نضيف `5850+` لمصاريف الشحن.
> في الشجرات القائمة، نضيف مجموعة `5750 مصاريف الشحنات والاستيراد`

## 🔄 2. الدورة المحاسبية الكاملة للكونتينر

> **⚡ المبدأ المحاسبي**: المصاريف تُرسمَل (تُضاف لقيمة الأصل) في حساب بضاعة بالطريق،
> والالتزام يُسجَّل في حساب المورد/مقدم الخدمة (من له المال على من).

### المرحلة ①: شراء من مورد (موجود ✅)
```
مد: 5200 المشتريات                         150,000 USD
    دا: 2110x حساب المورد (شركة الأقمشة)       150,000 USD
```
> الالتزام: نحن مدينون لشركة الأقمشة بـ 150,000

### المرحلة ②: تسجيل مصاريف الكونتينر (القيد الصحيح — رسملة + التزام)
```
مد: 1143x بضاعة بالطريق — كونتينر TEXU-001     9,000 USD  ← الأصل يزداد
    دا: 2110x حساب مورد الشحن (شركة الشحن)       5,000 USD  ← التزام لشركة الشحن
    دا: 2110x حساب مورد الجمارك (مصلحة الجمارك)   3,000 USD  ← التزام للجمارك
    دا: 2110x حساب مورد التأمين (شركة التأمين)    1,000 USD  ← التزام لشركة التأمين
```
> **التحليل المحاسبي:**
> - **المدين**: حساب الكونتينر (أصل متداول) — تكلفة البضاعة بالطريق تزداد
> - **الدائن**: حسابات الموردين/مقدمي الخدمة — التزامات علينا لهم
> - **لمن المال؟** لشركة الشحن والجمارك والتأمين (نحن مدينون لهم)

### المرحلة ③: الدفع من الصندوق (سندات صرف عادية)
```
مد: 2110x حساب مورد الشحن     5,000 USD  ← تسوية الالتزام
    دا: 1110 الصندوق / 1120 البنك   5,000 USD  ← خروج النقد
```
> يتم من خلال سندات الصرف العادية — كل مورد يُدفع على حدة

### المرحلة ④: استلام البضاعة وتوزيع المصاريف (مستقبلي)
```
مد: 1140 المخزون (+ توزيع التكلفة على المواد)   159,000 USD
    دا: 5200 المشتريات                             150,000 USD
    دا: 1143x بضاعة بالطريق — كونتينر TEXU-001     9,000 USD
```
> حساب الكونتينر يعود لصفر — البضاعة انتقلت للمستودع

### 📊 حركة حساب الكونتينر (دفتر الأستاذ):

| الحركة | مدين | دائن | الرصيد |
|--------|------|------|--------|
| ② تسجيل مصاريف | 9,000 | | 9,000 (مد) |
| ④ استلام بالمستودع | | 9,000 | 0 ✅ |

### المرحلة B: Trigger إنشاء حساب الكونتينر تلقائياً

```sql
CREATE OR REPLACE FUNCTION create_container_account()
RETURNS TRIGGER
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
    v_parent_id UUID;           -- 1143
    v_account_type_id UUID;     -- CURRENT_ASSET
    v_new_code VARCHAR(50);
    v_new_account_id UUID;
    v_full_code VARCHAR(200);
BEGIN
    -- الحصول على حساب بضاعة بالطريق الأب (1143)
    SELECT id INTO v_parent_id 
    FROM chart_of_accounts 
    WHERE company_id = NEW.company_id 
      AND account_code = '1143';
    
    IF v_parent_id IS NULL THEN
        RAISE NOTICE 'لا يوجد حساب بضاعة بالطريق (1143)، تخطي';
        RETURN NEW;
    END IF;
    
    -- الحصول على نوع الحساب
    SELECT id INTO v_account_type_id FROM account_types WHERE code = 'CURRENT_ASSET';
    
    -- توليد رقم الحساب الفرعي التلقائي
    v_new_code := generate_next_account_code(NEW.tenant_id, NEW.company_id, '1143');
    v_full_code := '1000.1100.1140.1143.' || v_new_code;
    
    -- إنشاء الحساب الفرعي
    INSERT INTO chart_of_accounts (
        tenant_id, company_id, account_code,
        name_ar, name_en,
        account_type_id, parent_id,
        is_group, is_detail, level, full_code,
        currency, is_active
    ) VALUES (
        NEW.tenant_id, NEW.company_id, v_new_code,
        'بضاعة بالطريق — ' || COALESCE(NEW.container_number, NEW.id::TEXT),
        'Goods in Transit — ' || COALESCE(NEW.container_number, NEW.id::TEXT),
        v_account_type_id, v_parent_id,
        false, true, 5, v_full_code,
        COALESCE(NEW.currency, 'USD'), true
    ) RETURNING id INTO v_new_account_id;
    
    -- ربط الحساب بالكونتينر
    NEW.container_account_id := v_new_account_id;
    
    RETURN NEW;
END;
$$;

-- Trigger
DROP TRIGGER IF EXISTS trg_create_container_account ON containers;
CREATE TRIGGER trg_create_container_account
    BEFORE INSERT ON containers
    FOR EACH ROW
    EXECUTE FUNCTION create_container_account();
```

// ===== القيد الصحيح: رسملة + التزام =====

// المدين الوحيد: حساب بضاعة بالطريق (الكونتينر)
const containerAccountId = data?.container_account_id;
if (!containerAccountId) {
    toast.error('لم يتم إنشاء حساب بضاعة بالطريق لهذا الكونتينر');

// سطر المدين: الكونتينر (الأصل يزداد)
const totalAmount = pendingExpenses.reduce((sum, e) => sum + e.amount, 0);
const journalLines = [{
    account_id: containerAccountId,
    debit: totalAmount,
    credit: 0,
    description: `رسملة مصاريف كونتينر ${data?.container_number}`,
}];

// سطور الدائن: حساب كل مورد/مقدم خدمة (الالتزام)
for (const expense of pendingExpenses) {
    journalLines.push({
        account_id: expense.vendor_account_id,  // حساب المورد في الشجرة

        credit: expense.amount,
        description: `${expense.expense_type}: ${expense.description}`,

}

## ⚠️ 5. اعتبارات مهمة

### 5.1 الشركات القائمة
- يجب تشغيل migration لتحويل 1143 من تفصيلي لمجموعة
- يجب إنشاء حسابات فرعية للكونتينرات الموجودة مسبقاً

### 5.2 تسلسل الترقيم
- `generate_next_account_code('1143')` يولد: 11431, 11432, 11433...
- الحد الأقصى: 9 فرعيات برقم واحد (يكفي لكل فترة)
- يمكن توسيعه لرقمين (1143xx) إذا لزم

### 5.3 العملة
- حساب الكونتينر يأخذ عملة الكونتينر (عادةً USD للشحنات الدولية)
- تحويل العملة يتم عند الاستلام والتوزيع

### 5.4 إقفال الحساب
- عند استلام كامل البضاعة، يجب أن يكون رصيد حساب الكونتينر = صفر
- أي رصيد متبقي يُرحَّل لفروقات

### 5.5 صحة القيد المحاسبي (من له المال على من)
- **مد (أصول ↑)**: حساب الكونتينر — البضاعة بالطريق تزداد قيمتها
- **دا (التزامات ↑)**: حسابات الموردين — نحن مدينون لهم بالمبلغ
- **الدفع لاحقاً**: سند صرف عادي يُقلل الالتزام ويُقلل الصندوق
