# خطة المصاريف الفعلية - نظام الدفعات (Batch Expenses)
> **تاريخ:** 2026-02-17  
> **الحالة:** جاري التنفيذ

---

## 🎯 الفكرة

المصاريف الفعلية تعمل كـ **نظام فواتير/دفعات**:

```
┌─────────────────────────────────────────────────────┐
│  الجلسة الأولى (مثلاً: دفعة الشحن)                 │
│                                                      │
│  + مصروف 1: مايرسك — شحن بحري    $3,000             │
│  + مصروف 2: وكيل جمركي           $500               │
│  + مصروف 3: تأمين بحري            $200               │
│                                                      │
│  [ 💾 حفظ وترحيل ] ← قيد واحد JE-0042:              │
│     مدين: 5811 مايرسك            $3,000              │
│     مدين: 5821 وكيل جمركي        $500                │
│     مدين: 5831 تأمين بحري         $200                │
│     دائن: حساب الكونتينر         $3,700              │
└─────────────────────────────────────────────────────┘

... بعد أسبوع ...

┌─────────────────────────────────────────────────────┐
│  الجلسة الثانية (مثلاً: دفعة الجمارك)              │
│                                                      │
│  + مصروف 4: رسوم جمركية          $1,200             │
│  + مصروف 5: رسوم ميناء            $150               │
│                                                      │
│  [ 💾 حفظ وترحيل ] ← قيد جديد JE-0058:             │
│     مدين: 5822 رسوم جمركية       $1,200             │
│     مدين: 5841 رسوم ميناء         $150               │
│     دائن: حساب الكونتينر         $1,350              │
└─────────────────────────────────────────────────────┘
```

### القواعد:
1. **إضافة واحد أو أكثر** من المصاريف في نفس الجلسة
2. **الحفظ = قيد محاسبي واحد** لكل دفعة
3. **فتح دفعة جديدة** في أي وقت ← قيد جديد
4. **عرض الدفعات السابقة** مع تفاصيل القيد

---

## 📋 مراحل التنفيذ

### المرحلة 1: إصلاح الخطأ + القائمة المؤقتة ⬅️ الأولوية

**الهدف:** المستخدم يضيف مصاريف متعددة (state فقط) قبل الحفظ

**الخطوات:**
1. إصلاح خطأ `currency` → استخدم `currency_code` فقط
2. تعديل `newActual` state ليستخدم `currency_code` بدل `currency`
3. إضافة `pendingExpenses: ActualExpenseRow[]` — قائمة مؤقتة (لم تُحفظ بعد)
4. زر "إضافة" يضيف سطر للقائمة المؤقتة (لا يحفظ في DB)
5. عرض القائمة المؤقتة بجدول بسيط مع ❌ حذف لكل سطر
6. عرض إجمالي الدفعة

### المرحلة 2: الحفظ الجماعي + القيد الموحّد

**الهدف:** حفظ كل المصاريف المعلقة + إنشاء قيد واحد

**الخطوات:**
1. زر **"💾 حفظ وترحيل الدفعة"** يفعل:
   - حفظ كل المصاريف في `container_expenses` (bulk insert)
   - بناء القيد المحاسبي:
     - **مدين**: سطر لكل حساب مصروف
     - **مدين إضافي**: ضريبة مدخلات (مجموع كل الضرائب)
     - **دائن**: حساب الكونتينر (= مجموع المدين)
   - ترحيل القيد فوراً
   - ربط `journal_entry_id` بكل المصاريف
   - تصفير القائمة المعلقة

### المرحلة 3: عرض الدفعات المحفوظة

**الهدف:** عرض الدفعات السابقة مجمّعة حسب `journal_entry_id`

**الخطوات:**
1. تجميع المصاريف حسب `journal_entry_id` → كل مجموعة = دفعة
2. عرض كل دفعة كبطاقة/كارد قابلة للطي:
   - رأس: رقم القيد + التاريخ + الإجمالي
   - تفاصيل: جدول المصاريف (نوع + حساب + مبلغ)
3. زر عرض القيد المحاسبي

### المرحلة 4: تعديل/حذف (لاحقاً)

- تعديل دفعة = عكس القيد القديم + إنشاء قيد جديد
- حذف دفعة = عكس القيد + حذف المصاريف

## 🗄️ البنية في DB

```
container_expenses:
  id, container_id, tenant_id, company_id
  expense_type, description, vendor_name
  amount, actual_amount, amount_before_tax
  tax_rate, tax_amount
  currency_code,                    ← ✅ (ليس currency!)
  exchange_rate
  expense_account_id               ← حساب المصروف/المورد
  journal_entry_id                 ← يربط كل مصاريف الدفعة بنفس القيد
  is_posted                        ← true بعد الترحيل
  ...
```

---

## 📐 القيد المحاسبي الموحّد

```
مثال: دفعة بـ 3 مصاريف + ضريبة 15% على المصروف الأول فقط

مدين  5811  مايرسك — شحن بحري        $3,000
مدين  1160  ضريبة مدخلات               $450    ← 15% × 3000
مدين  5821  وكيل جمركي                $500    ← بدون ضريبة
مدين  5831  تأمين بحري                 $200    ← بدون ضريبة
  دائن  حساب الكونتينر               $4,150  ← المجموع
```

## ⏳ ترتيب التنفيذ

| المرحلة | الوصف | التقدير |
|---------|-------|---------|
| **M1** | إصلاح currency + القائمة المؤقتة | 🔄 الآن |
| **M2** | الحفظ الجماعي + القيد الموحّد | بعد M1 |
| **M3** | عرض الدفعات السابقة | بعد M2 |
| **M4** | تعديل/حذف | بعد M3 |
