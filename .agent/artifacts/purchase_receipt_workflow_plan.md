## 1. نوع الشراء (receipt_mode) ✅ مكتمل
- عمود `receipt_mode` تم إضافته (`direct` | `international`)
- Toggle في واجهة الشيت يعمل (أخضر=محلي، أزرق=دولي)
- الحفظ والاسترجاع يعمل
- **تبديل العملة التلقائي**: محلي→عملة الشركة، دولي→USD→EUR→عملة الشركة

## 2. عرض المستندات بانتظار الاستلام ✅ مكتمل
- تبويب "استلام بضائع" يعرض الآن:
  - إذون الاستلام الفعلية (purchase_receipts)
  - 🟠 أوامر شراء محلية مؤكدة (بحالة "غير مستلم")
  - 🟠 فواتير مشتريات محلية مؤكدة (بحالة "غير مستلم")
- بادج "بانتظار الاستلام (أمر/فاتورة)" برتقالي
- حالة "غير مستلم" بنقطة حمراء
- النقر يفتح المستند الأصلي في وضع المعاينة

## 3. Tooltips مساعدة ✅ مكتمل

## 4. إذن الاستلام المخصص للأقمشة 🔜 قادم — أولوية عالية

### الفكرة:
إذن الاستلام في نظام الأقمشة يختلف عن الأنظمة التقليدية لأنه يتعامل مع **رولونات** (Rolls) وليس فقط كميات.

### سير العمل المقترح:
```
فاتورة/أمر محلي مؤكد (بانتظار الاستلام)
    ↓
أمين المستودع يضغط "بدء الاستلام"
    ↓
شاشة استلام مخصصة للأقمشة:
├── عرض بنود الفاتورة (المواد المطلوبة)
├── لكل مادة: إدخال الرولونات المستلمة
│   ├── رقم الرولون (Roll #) — تسلسلي أو يدوي
│   ├── الطول/الوزن الفعلي (actual length/weight)
│   ├── العرض (width)
│   ├── اللون/الدرجة (color/shade)
│   ├── ملاحظات جودة
│   └── [إمكانية مسح باركود]
├── مقارنة: الكمية المطلوبة vs الكمية المستلمة
└── تسجيل الفروقات (نقص/زيادة/تالف)
    ↓
تأكيد الاستلام → إنشاء fabric_rolls تلقائياً
    ↓
تحديث المخزون (المخزون بالرولونات + الكمية الإجمالية)
    ↓
حالة الفاتورة: "مستلمة" أو "مستلمة جزئياً"
```

### جداول قاعدة البيانات المتوقعة:
```sql
-- إذن الاستلام
purchase_receipts (موجود بالفعل)
  + source_order_id  → ربط بأمر الشراء
  + source_invoice_id → ربط بفاتورة المشتريات
  + receipt_type: 'direct' | 'shipment'

-- بنود الاستلام (المواد)
purchase_receipt_items
  + material_id
  + quantity_ordered (الكمية المطلوبة)
  + quantity_received (الكمية المستلمة الفعلية)
  + quantity_damaged (الكمية التالفة)
  + quantity_shortage (النقص)

-- الرولونات المستلمة (مخصص للأقمشة)
fabric_rolls (موجود بالفعل)
  + receipt_item_id → ربط ببند الاستلام
  + roll_number
  + actual_length / actual_weight
  + width
  + color_shade
  + quality_notes
  + barcode
```

### واجهة المستخدم المقترحة:
1. **شاشة استلام مخصصة** (ليست الشيت العادي) — تصميم مخصص لسرعة الإدخال
2. **جدول رولونات** قابل للتوسيع لكل مادة
3. **ملخص مباشر**: المطلوب | المستلم | الفرق
4. **زر "إضافة رولون سريع"** — يولد رقم تسلسلي تلقائياً
5. **دعم الباركود** (مستقبلي) — مسح بكاميرا الموبايل عبر TexaMobile

## 5. صلاحيات الاستلام 🔜 قادم

### المنطق:
| الدور | المعاينة | إنشاء إذن استلام | تأكيد الاستلام |
|-------|---------|------------------|---------------|
| مستخدم عادي | ✅ فقط معاينة | ❌ | ❌ |
| أمين مستودع | ✅ | ✅ (مستودعه فقط) | ✅ |
| مدير | ✅ | ✅ (كل المستودعات) | ✅ |

### حالات الاستلام (receipt_status):
| الحالة | المعنى | اللون |
|--------|--------|-------|
| `not_received` | غير مستلم | 🔴 أحمر |
| `partially_received` | مستلم جزئياً | 🟡 أصفر |
| `received` | مستلم بالكامل | 🟢 أخضر |

### التعديلات المطلوبة:
1. **صلاحيات RLS**: أمين المستودع يرى فقط الفواتير المخصصة لمستودعه
2. **إخفاء زر الاستلام**: للمستخدمين العاديين
3. **إشعارات**: لأمين المستودع عند تأكيد فاتورة محلية جديدة

## 6. سير عمل الاستلام للمشتريات الدولية 🔜 قادم

### المنطق:
```
فاتورة دولية (international) + مؤكدة (confirmed)
    ↓
تظهر عند إضافة فواتير للكونتينر
    ↓
الكونتينر يمر بمراحل الشحن والتخليص
    ↓
عند وصول الكونتينر → إنشاء إذن استلام جماعي
    ↓
أمين المستودع يستلم البضائع بناءً على الكونتينر
    ↓
نفس شاشة استلام الأقمشة (رولونات) تُستخدم هنا أيضاً
```
