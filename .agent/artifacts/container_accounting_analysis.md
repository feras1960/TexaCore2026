### الكونتينر: MCKUM3253424
| البند | المبلغ | الحالة |
|-------|--------|--------|
| فاتورة مشتريات (نانسي) | 16,900 USD | ❌ ذهبت لـ COGS مباشرة بدلاً من حساب الكونتينر |

| ضريبة جمركية | 845 USD | ✅ مرحّلة لحساب الكونتينر |

### المشاكل المكتشفة:

#### 🔴 مشكلة 1: فاتورة المشتريات لم تُرحّل لحساب الكونتينر
- **السبب:** حقل `container_account_id` في جدول `containers` **فارغ (NULL)**
- **النتيجة:** الكود يتراجع (fallback) لحساب تكلفة الأقمشة المباعة (COGS) مباشرة
- **الكود المسؤول:** `purchaseAccountingService.ts` سطر 261-278

#### 🔴 مشكلة 2: قيد الإقفال ناقص
- **السبب:** دالة `finalizeLandedCost` تحتوي `// TODO: إنشاء القيد المحاسبي النهائي`
- **النتيجة:** قيد الإقفال يحتوي فقط على المصاريف (14,745) بدون قيمة البضاعة (16,900)
- **المبلغ الصحيح:** 30,800 USD = بضاعة (16,900) + مصاريف (13,900) + أو بدون ضريبة حسب نوعها

#### 🔴 مشكلة 3: لا يوجد حساب كونتينر تلقائي
- عند إنشاء كونتينر جديد، لا يتم إنشاء حساب فرعي تلقائياً في شجرة الحسابات

## 🔧 خطة الإصلاح

### الإصلاح 1: إنشاء حساب كونتينر تلقائياً
**الملف:** `containersService.ts` → `createContainer()`

عند إنشاء كونتينر جديد:
1. البحث عن حساب "بضاعة بالطريق" (1143) كحساب أب
2. إنشاء حساب فرعي بالرقم: `1143-{container_number}`
3. حفظ `container_account_id` في جدول `containers`

```typescript
// عند إنشاء كونتينر:
// 1. إيجاد الحساب الأب (1143 بضاعة بالطريق)
// 2. إنشاء حساب فرعي: "كونتينر MCKUM3253424"
// 3. ربطه بالكونتينر container_account_id = new_account.id
```

### الإصلاح 2: فاتورة المشتريات → حساب الكونتينر
**الملف:** `purchaseAccountingService.ts`

- **الكود موجود بالفعل** ✅ (سطر 261-278)
- يحتاج فقط أن `container_account_id` يكون مملوء ← يُحل بالإصلاح 1

### الإصلاح 3: مصاريف الكونتينر → حساب الكونتينر
**الملف:** `containersService.ts` → `createExpenseJournalEntry()`

- حالياً: المستخدم يختار حساب المدين يدوياً
- **المطلوب:** الحساب المدين يكون **تلقائياً = حساب الكونتينر**
- هذا يجمع كل التكاليف في حساب واحد

### الإصلاح 4: قيد الإقفال الصحيح
**الملف:** `containersService.ts` → `finalizeLandedCost()`

عند إقفال الكونتينر:
```
مدين │ 1141 بضاعة جاهزة   │ Total Landed Cost (FOB + expenses)
دائن │ 1432 حساب كونتينر   │ Total Landed Cost

► يُصفّر حساب الكونتينر
► البضاعة تدخل المخزون بالتكلفة الحقيقية (Landed Cost)
```

### الإصلاح 5: إخفاء القيود الآلية
- القيود الآلية (النظام) تُنشأ بـ `entry_type = 'system'` أو `is_system = true`
- في القيود اليومية: فلتر افتراضي يُخفي قيود النظام
- زر "إظهار قيود النظام" يعرضها عند الحاجة

## ⚡ ملاحظات مهمة

1. **حساب الكونتينر مؤقت** — يُنشأ عند إنشاء الكونتينر ويُصفّر عند الإقفال
2. **COGS لا تُستخدم إلا عند البيع** — وليس عند الشراء
3. **الضريبة المستردة** (VAT) لا تدخل تكلفة الكونتينر — تذهب لحساب ضريبة المدخلات
4. **القيود الآلية** يجب أن تكون مخفية عن المستخدم بشكل افتراضي
5. **التوزيع** (Allocation) يحدد تكلفة كل صنف = (FOB + حصته من المصاريف)
