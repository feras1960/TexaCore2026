# وورك فلو المبيعات (Sales Workflow)

## المسار الكامل (Full Pipeline)
```
عرض سعر → حجز بضائع → أمر بيع → إذن تسليم → فاتورة مبيعات → مكتمل
Quotation → Reservation → Sales Order → Delivery Note → Sales Invoice → Completed
```

## قواعد العمل (Business Rules)

### 1. عرض سعر (Quotation)
- **الحالة**: draft → sent → approved → rejected → expired
- **الصلاحية**: أي موظف مبيعات
- **التحويل**: يمكن تحويله إلى:
  - أمر بيع (مباشرة)
  - حجز بضائع (إذا العميل يريد حجز قبل الشراء)

### 2. حجز بضائع (Reservation)
- **الحالة**: draft → confirmed → released → cancelled
- **الصلاحية**: موظف مبيعات + أمين مستودع (للتأكيد)
- **التأثير**: يحجز الكمية من المخزون (لا يخصمها)
- **التحويل**: يمكن تحويله إلى أمر بيع

### 3. أمر بيع (Sales Order)
- **الحالة**: draft → confirmed → processing → completed → cancelled
- **الصلاحية**: موظف مبيعات
- **التحويل**: يمكن تحويله إلى إذن تسليم

### 4. إذن تسليم (Delivery Note)
- **الحالة**: draft → picking → packed → shipped → delivered → cancelled
- **الصلاحية**: أمين المستودع حصراً
- **التأثير عند التنفيذ**: 
  - يخصم الكمية من المخزون
  - يُنشئ فاتورة مبيعات تلقائياً
- **المتطلبات**: يتطلب أمر بيع مرتبط (source_order_id)

### 5. فاتورة مبيعات (Sales Invoice)
- **الحالة**: draft → posted → paid → cancelled
- **كيف تُنشأ**:
  - **طريقة 1 (تلقائية)**: من تنفيذ إذن التسليم
  - **طريقة 2 (POS - بيع مباشر من المتجر)**: 
    - الموظف ينشئ فاتورة مباشرة
    - يختار بضائع من مخزون المتجر/الفرع
    - عند الحفظ: يُنفذ إذن التسليم تلقائياً + يخصم المخزون
  - **طريقة 3 (أدمن)**: الأدمن يمكنه إنشاء فاتورة مباشرة

## سيناريو البيع من المتجر (POS Mode)
1. الموظف يفتح فاتورة مبيعات جديدة
2. يحدد المتجر/الفرع (warehouse_id)
3. يضيف البضائع (يتحقق من الكمية المتاحة في المتجر)
4. يحفظ الفاتورة
5. النظام يقوم تلقائياً بـ:
   a. إنشاء إذن تسليم (status = 'delivered')
   b. خصم الكمية من مخزون المتجر
   c. تحديث الفاتورة (status = 'posted')

## الصلاحيات (Permissions)
| الإجراء | موظف مبيعات | أمين مستودع | مدير/أدمن |
|---------|-------------|-------------|-----------|
| عرض سعر | ✅ | ❌ | ✅ |
| حجز بضائع | ✅ (إنشاء) | ✅ (تأكيد) | ✅ |
| أمر بيع | ✅ | ❌ | ✅ |
| إذن تسليم | ❌ | ✅ | ✅ |
| فاتورة (POS) | ✅ (من متجره) | ❌ | ✅ |
| فاتورة (عادية) | ❌ | ❌ | ✅ |

## الربط مع الكانبان
- السحب بين الأعمدة = تحويل نوع المستند
- التحويل يتبع القواعد أعلاه
- لا يمكن القفز فوق مراحل (باستثناء POS)
- لا يمكن الرجوع للخلف

## الجداول المتأثرة
- `sales_orders` - أوامر البيع
- `sales_invoices` - فواتير المبيعات  
- `quotations` - عروض الأسعار
- `delivery_notes` - أذونات التسليم
- `inventory_transactions` - حركات المخزون
- `stock_levels` / `inventory_items` - مستويات المخزون
