## ✅ 4. منطق ترحيل فاتورة المشتريات (`purchaseAccountingService`)

### القيد المحاسبي عند الترحيل:
```
مدين: حساب المشتريات (511) أو حساب الكونتينر (11432)
مدين: ضريبة المدخلات (إن وجدت)
دائن: دين الموردين (2111) + party_type='supplier' + party_id=supplier_id
```

### مصدر الحسابات:
1. **supplier.payable_account_id** → حساب الذمم الخاص بالمورد ✅
2. **Fallback**: `defaults.default_payable_account_id` من الإعدادات ✅
3. **party_type + party_id** مربوطين بالمورد → Sub-Ledger ✅

### فاتورة كونتينر:
- إذا `invoice.container_id` موجود → **المدين = حساب الكونتينر** (بضاعة بالطريق) ✅
- إذا لا → **المدين = حساب المشتريات/تكلفة البضاعة** ✅
