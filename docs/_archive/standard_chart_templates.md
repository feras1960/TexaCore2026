## 🚢 آلية حسابات الكونتينرات (ديناميكية)

> **مهم:** حساب `1143 بضاعة في الطريق` هو **مجموعة** — عند إنشاء كل كونتينر جديد
> يُنشأ **حساب فرعي تلقائياً** تحت هذه المجموعة.

### مثال حي:
```
1143  بضاعة في الطريق              Goods in Transit            [مجموعة] 🚢
├── 11431  كونتينر TEXU-2024-001   Container TEXU-2024-001     [تفصيلي] ← ينشأ تلقائياً
├── 11432  كونتينر MSKU-2024-015   Container MSKU-2024-015     [تفصيلي] ← ينشأ تلقائياً
├── 11433  كونتينر FERU-2024-008   Container FERU-2024-008     [تفصيلي] ← ينشأ تلقائياً
└── ...    (حساب لكل كونتينر جديد)
```

### آلية العمل:
1. **عند إنشاء كونتينر جديد** → يُنشأ حساب فرعي تحت `1143` تلقائياً
2. **اسم الحساب** = `كونتينر` + رقم الكونتينر (عربي/إنجليزي)
3. **رمز الحساب** = `1143` + رقم تسلسلي (11431, 11432, 11433...)
4. **كل مصاريف الكونتينر** تُرسمل على حسابه الخاص (مدين)
5. **عند استلام البضاعة** → يُحوّل الرصيد من حساب الكونتينر إلى المخزون

### القيد المحاسبي لمصاريف الكونتينر:
```
مدين  11431  كونتينر TEXU-2024-001    $3,000  (رسملة — أصل)
  دائن  2111  المورد / شركة الشحن      $3,000  (التزام)
```

### القيد عند استلام البضاعة (تحويل من بالطريق → مخزون):
```
مدين  1141  بضاعة جاهزة              $3,000  (دخول المخزون)
  دائن  11431  كونتينر TEXU-2024-001  $3,000  (تصفية الكونتينر)
```

### ⚠️ ملاحظة: هذا متاح في الموسعة والموسعة أقمشة فقط — القياسية لا تدعم كونتينرات
