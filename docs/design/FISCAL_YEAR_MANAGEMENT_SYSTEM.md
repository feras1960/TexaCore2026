## 1. الفكرة الأساسية

### 💡 المفهوم:

> **منح المدير الرئيسي (Super Admin/Owner) القدرة على اختيار أسلوب الترابط بين السنوات المالية حسب طبيعة عمل الشركة.**

```
┌─────────────────────────────────────────────────────────────────────┐
│                    إعدادات السنوات المالية                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  اختر أسلوب إدارة السنوات المالية:                                   │
│                                                                      │
│  ○ السنوات المستقلة (Independent Years)                             │
│    └── كل سنة مُغلقة تُعتبر قاعدة بيانات منفصلة                      │
│    └── التعديل والحذف متاح داخل السنة المُغلقة                       │
│    └── مناسب للشركات الصغيرة والمتوسطة                               │
│                                                                      │
│  ○ السنوات المترابطة (Linked Years)                                 │
│    └── السنوات مُرتبطة محاسبياً                                      │
│    └── لا تعديل على السنوات المُغلقة                                 │
│    └── التصحيح بقيد تسوية في السنة الحالية                           │
│    └── مناسب للشركات المُلزمة ضريبياً/مُراجعة خارجياً               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. النظامان المتاحان

### 🔹 النظام 1: السنوات المستقلة (Independent Years)

```
┌─────────────────────────────────────────────────────────────────────┐
│  📦 السنة 2024 (مُغلقة)       📦 السنة 2025 (مُغلقة)                  │
│  ┌────────────────────┐      ┌────────────────────┐                  │
│  │ قيود 2024          │      │ قيود 2025          │                  │
│  │ فواتير 2024        │      │ فواتير 2025        │                  │
│  │ ختامي 2024         │      │ ختامي 2025         │                  │
│  └────────────────────┘      └────────────────────┘                  │
│         ↑ مستقلة                    ↑ مستقلة                         │
│         لا ارتباط ←────────────────→ لا ارتباط                       │
│                                                                      │
│  📦 السنة 2026 (الحالية - مفتوحة)                                    │
│  ┌────────────────────┐                                              │
│  │ الأرصدة الافتتاحية │ ← تم نقلها عند الإغلاق                       │
│  │ قيود 2026          │                                              │
│  └────────────────────┘                                              │
│                                                                      │
├─────────────────────────────────────────────────────────────────────┤
│  ✅ المسموح في السنة المُغلقة:                                       │
│  ├── تعديل القيود ✅                                                 │
│  ├── تعديل الفواتير ✅                                               │
│  ├── حذف (مع Audit Log) ⚠️                                           │
│  └── قيد تسوية في السنة الحالية ✅                                   │
│                                                                      │
│  ⚠️ ملاحظة مهمة:                                                     │
│  التعديلات لا تؤثر على الأرصدة الافتتاحية للسنة التالية             │
│  (تم نقلها عند الإغلاق وأصبحت ثابتة)                                 │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**مناسب لـ:**
- الشركات الصغيرة والمتوسطة
- الشركات غير المُلزمة بالمراجعة الخارجية
- الشركات التي تحتاج مرونة في التصحيحات

### 🔹 النظام 2: السنوات المترابطة (Linked Years)

```
┌─────────────────────────────────────────────────────────────────────┐
│  📦 السنة 2024 (مُغلقة)                                              │
│  ┌────────────────────┐                                              │
│  │ قيود 2024          │ ← 🔒 لا تعديل                                │
│  │ ختامي 2024         │                                              │
│  │ أرباح مُرحّلة      │ ────────────────────┐                        │
│  └────────────────────┘                     │                        │
│                                             ↓                        │
│  📦 السنة 2025 (مُغلقة)                                              │
│  ┌────────────────────┐                                              │
│  │ أرصدة افتتاحية     │ ← من 2024                                    │
│  │ قيود 2025          │ ← 🔒 لا تعديل                                │
│  │ ختامي 2025         │                                              │
│  │ أرباح مُرحّلة      │ ────────────────────┐                        │
│  └────────────────────┘                     │                        │
│                                             ↓                        │
│  📦 السنة 2026 (الحالية - مفتوحة)                                    │
│  ┌────────────────────┐                                              │
│  │ أرصدة افتتاحية     │ ← من 2025                                    │
│  │ قيود 2026          │ ← تعديل حر                                   │
│  │ قيود تسوية         │ ← لتصحيح السنوات السابقة                     │
│  └────────────────────┘                                              │
│                                                                      │
├─────────────────────────────────────────────────────────────────────┤
│  ✅ المسموح في السنة المُغلقة:                                       │
│  ├── تعديل ❌ (ممنوع)                                                │
│  ├── حذف ❌ (ممنوع)                                                  │
│  └── قيد تسوية في السنة الحالية ✅                                   │
│                                                                      │
│  📋 قيد التسوية:                                                     │
│  ├── يُنشأ في السنة الحالية                                          │
│  ├── يرتبط بالقيد الأصلي (reference_id)                              │
│  ├── يُوضّح أنه "تصحيح سنة سابقة"                                    │
│  └── يُسجّل في Audit Log                                             │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**مناسب لـ:**
- الشركات المُلزمة ضريبياً
- الشركات الخاضعة للمراجعة الخارجية
- الشركات الكبيرة
- الشركات في قطاع البنوك/التأمين

## 3. التصميم التقني

### 📊 هيكل قاعدة البيانات:

```sql
-- ═══════════════════════════════════════════════════════════════
-- إضافة إعدادات السنوات المالية للشركة
-- ═══════════════════════════════════════════════════════════════

-- الخيار 1: إضافة للجدول الموجود
ALTER TABLE companies ADD COLUMN IF NOT EXISTS 
    fiscal_year_mode VARCHAR(20) DEFAULT 'independent'
    CHECK (fiscal_year_mode IN ('independent', 'linked'));

ALTER TABLE companies ADD COLUMN IF NOT EXISTS 
    accounting_settings JSONB DEFAULT '{
        "fiscal_year_mode": "independent",
        "allow_edit_closed_year": true,
        "allow_delete_closed_year": false,
        "require_adjustment_entry": true,
        "auto_link_adjustments": true,
        "require_edit_reason": true,
        "notify_on_closed_year_edit": true
    }';

-- ═══════════════════════════════════════════════════════════════
-- جدول قيود التسوية/التصحيح
-- ═══════════════════════════════════════════════════════════════

CREATE TABLE IF NOT EXISTS adjustment_entries (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
    company_id UUID NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
    
    -- القيد الجديد (التسوية)
    new_entry_id UUID NOT NULL REFERENCES journal_entries(id),
    
    -- القيد الأصلي المُصحح
    original_entry_id UUID REFERENCES journal_entries(id),
    original_entry_date DATE,
    original_fiscal_year_id UUID REFERENCES fiscal_years(id),
    
    -- تفاصيل التصحيح
    adjustment_type VARCHAR(30) NOT NULL, -- 'correction', 'reclassification', 'reversal'
    reason TEXT NOT NULL,
    
    -- الفروقات
    amount_difference DECIMAL(18,2),
    account_changes JSONB, -- {old_account, new_account, amount}
    
    -- الموافقات
    created_by UUID REFERENCES user_profiles(id),
    approved_by UUID REFERENCES user_profiles(id),
    approved_at TIMESTAMPTZ,
    
    created_at TIMESTAMPTZ DEFAULT NOW(),
    
    UNIQUE(new_entry_id)
);

-- ═══════════════════════════════════════════════════════════════
-- دالة التحقق من إمكانية التعديل
-- ═══════════════════════════════════════════════════════════════

CREATE OR REPLACE FUNCTION can_edit_entry(
    p_entry_id UUID,
    p_user_id UUID DEFAULT auth.uid()
)
RETURNS JSONB AS $$
DECLARE
    v_entry RECORD;
    v_fiscal_year RECORD;
    v_company RECORD;
    v_settings JSONB;
    v_result JSONB;
BEGIN
    -- الحصول على القيد
    SELECT * INTO v_entry FROM journal_entries WHERE id = p_entry_id;
    IF NOT FOUND THEN
        RETURN jsonb_build_object('can_edit', false, 'reason', 'entry_not_found');
    END IF;
    
    -- الحصول على السنة المالية
    SELECT * INTO v_fiscal_year FROM fiscal_years WHERE id = v_entry.fiscal_year_id;
    
    -- الحصول على إعدادات الشركة
    SELECT accounting_settings INTO v_settings FROM companies WHERE id = v_entry.company_id;
    
    -- إذا كانت السنة مفتوحة
    IF v_fiscal_year.status = 'open' THEN
        RETURN jsonb_build_object(
            'can_edit', true,
            'mode', 'direct',
            'reason', 'year_open'
        );
    END IF;
    
    -- السنة مُغلقة - نتحقق من الإعدادات
    IF v_settings->>'fiscal_year_mode' = 'independent' THEN
        -- النظام المستقل: يمكن التعديل
        RETURN jsonb_build_object(
            'can_edit', true,
            'mode', 'direct_with_warning',
            'reason', 'independent_mode',
            'warning', 'السنة المالية مُغلقة. التعديل لن يؤثر على السنوات اللاحقة.',
            'options', jsonb_build_array(
                jsonb_build_object('id', 'edit', 'label', 'تعديل مباشر'),
                jsonb_build_object('id', 'adjustment', 'label', 'قيد تسوية في السنة الحالية')
            )
        );
    ELSE
        -- النظام المترابط: قيد تسوية فقط
        RETURN jsonb_build_object(
            'can_edit', false,
            'mode', 'adjustment_only',
            'reason', 'linked_mode',
            'message', 'السنة المالية مُغلقة ومترابطة. يمكنك إنشاء قيد تسوية في السنة الحالية.',
            'options', jsonb_build_array(
                jsonb_build_object('id', 'adjustment', 'label', 'إنشاء قيد تسوية')
            )
        );
    END IF;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

## 5. سيناريوهات الاستخدام

### السيناريو 1: شركة صغيرة (سنوات مستقلة)

```
الشركة: مكتب محاماة صغير
النظام: السنوات المستقلة ✅

1. المحاسب يكتشف خطأ في فاتورة 2025
2. يفتح الفاتورة ويضغط "تعديل"
3. النظام يعرض:
   ├── ⚠️ هذه السنة مُغلقة
   ├── ◉ تعديل مباشر
   └── ○ قيد تسوية
4. المحاسب يختار "تعديل مباشر"
5. يُدخل سبب التعديل
6. يحفظ
7. ✅ تم التعديل مع Audit Log كامل
```

### السيناريو 2: شركة كبيرة (سنوات مترابطة)

```
الشركة: شركة مُساهمة مُلزمة بالتدقيق
النظام: السنوات المترابطة ✅

1. المحاسب يكتشف خطأ في قيد 2025
2. يفتح القيد ويضغط "تعديل"
3. النظام يعرض:
   ├── ⚠️ السنة مُغلقة ومترابطة
   ├── ❌ لا يمكن التعديل المباشر
   └── ✅ إنشاء قيد تسوية في 2026
4. المحاسب يضغط "إنشاء قيد تسوية"
5. النظام يفتح نموذج قيد جديد:
   ├── النوع: تسوية/تصحيح سنة سابقة
   ├── مرتبط بـ: JV-2025-1234
   └── التاريخ: اليوم (2026)
6. المحاسب يُدخل التسوية
7. ✅ تم إنشاء القيد مع الربط الكامل
```
