# هيكلية وتنفيذ وحدة المدفوعات (Payments Module Implementation)

📅 **تاريخ التحديث:** 2026-02-08
🛠️ **الحالة:** تم التنفيذ والتحقق (Verified)

## 1. نظرة عامة (Overview)
تم إنشاء وحدة مدفوعات متكاملة تدعم الربط مع كافة الكيانات التجارية والمالية في النظام. تعتمد الوحدة على جدول مركزي `payment_vouchers` لإدارة سندات الصرف والقبض، وتوفر واجهة موحدة `PaymentsList` لعرض وإدارة هذه السندات.

## 2. هيكلية قاعدة البيانات (Database Schema)

### الجدول الرئيسي: `payment_vouchers`
تم اعتماد الجدول الحالي `payment_vouchers` مع إجراء تحسينات هيكلية لدعم الربط المتعدد.

**الأعمدة الرئيسية:**
| العمود | النوع | الوصف | علاقة الربط (FK) |
| :--- | :--- | :--- | :--- |
| `id` | UUID | المعرف الفريد | PK |
| `voucher_date` | Date | تاريخ السند | - |
| `amount` | Numeric | المبلغ | - |
| `payment_method` | Varchar | طريقة الدفع (نقد، بنك، شيك) | - |
| `payment_number` | Varchar | رقم السند المرجعي | - |
| `company_id` | UUID | معرف الشركة (للتعددية) | `companies.id` |

**أعمدة الربط (Relations):**
تم إضافة مفاتيح أجنبية لربط السند بمختلف الكيانات:
1.  **`supplier_id`**: ربط مع الموردين (`suppliers`). تم إصلاح مشكلة عدم وجود قيد (Constraint) وتم تفعيله بنجاح.
2.  **`shipment_id`**: ربط مع الشحنات/الكونتينرات (`shipments`). يتيح تسجيل مصاريف على كونتينر محدد.
3.  **`purchase_invoice_id`**: ربط مع فواتير المشتريات (`purchase_invoices`). لسداد فاتورة محددة.
4.  *(ملاحظة: `customer_id` موجود ولكنه غير مفعل بالكامل في واجهة المدفوعات الحالية لأن التركيز على المدفوعات الخارجة للموردين)*.

## 3. التحديثات والتحسينات (Implemented Fixes)

### أ. إصلاحات قاعدة البيانات (Database Fixes)
1.  **إضافة الأعمدة المفقودة:** تم استخدام سكريبت `FIX_PAYMENT_VOUCHERS_SCHEMA.sql` لإضافة `shipment_id` و `purchase_invoice_id` و `payment_number`.
2.  **إصلاح القيود (FK Constraints):** تم حل خطأ `PGRST200` الذي كان يمنع جلب بيانات الموردين، وذلك بإضافة قيد `FOREIGN KEY` صريح للعمود `supplier_id` عبر سكريبت `FIX_PAYMENT_VOUCHERS_FK.sql`.
3.  **تأكيد الأمان (RLS):** تم تطبيق سياسات الوصول بناءً على `company_id` لضمان عزل البيانات بين الشركات.

### ب. تحديثات الواجهة (Frontend Updates)
1.  **تحديث `PaymentsList.tsx`:**
    *   تعديل الاستعلام ليقرأ من جدول `payment_vouchers` بدلاً من `payments` (الذي كان غير موجود).
    *   تحديث الجلب (Select Query) ليشمل العلاقات:
        ```typescript
        .select(`
            *,
            supplier:suppliers(name_ar, name_en),
            shipment:shipments(shipment_number),
            invoice:purchase_invoices(invoice_number)
        `)
        ```
    *   تحسين عرض الأعمدة ليشمل:
        *   **المورد (Payee):** يعرض الاسم العربي أو الإنجليزي للمورد.
        *   **المشروع/الكونتينر (Project/Container):** يعرض رقم الشحنة أو رقم الفاتورة المرتبطة بالسند، مما يسهل التتبع.

### ج. الترجمة (Localization)
تم التحقق من كافة مفاتيح الترجمة وإضافة الناقص منها في `ar.json` و `en.json`:
*   `actions.addExpense`: إضافة مصروف.
*   `types.dry`, `types.reefer`: أنواع الكونتينرات.
*   `messages.contentComingSoon` وغيرها.
