## 1. نظرة عامة {#1-نظرة-عامة}

### 🎯 الهدف
ربط جميع حركات المخزون بالنظام المحاسبي تلقائياً لضمان:
- دقة تكلفة المخزون
- صحة البيانات المالية
- التوازن المحاسبي الدائم

### 📊 مبدأ العمل
```
┌─────────────────────────────────────────────────────────────┐
│                    WAREHOUSE OPERATION                       │
│              (استلام / بيع / مناقلة / جرد)                    │
└─────────────────────┬───────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                   TRIGGER / FUNCTION                         │
│              (postgresql trigger أو RPC)                     │
└─────────────────────┬───────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                   JOURNAL ENTRY                              │
│              (قيد محاسبي تلقائي)                              │
│                                                              │
│   ┌─────────────────┐      ┌─────────────────┐              │
│   │   مدين (Debit)   │      │  دائن (Credit)  │              │
│   └─────────────────┘      └─────────────────┘              │
```

## 3. القيود المحاسبية التلقائية {#3-القيود-التلقائية}

### 📥 3.1 استلام بضاعة من المورد

**الحدث**: استلام رولونات من كونتينر أو فاتورة شراء

```sql
-- القيد المحاسبي
-- ═══════════════════════════════════════

مدين: 1403 - مخزون الأقمشة .............. 10,000 دولار
      دائن: 2101 - موردو الأقمشة .............. 10,000 دولار

-- الوصف: استلام 50 رولون قماش من المورد XYZ
-- المرجع: Container #CN-2026-001
```

**الجدول المتأثر**: `fabric_rolls`, `roll_movements`

**نوع الحركة**: `receive`

---

### 📤 3.2 بيع بضاعة للعميل

**الحدث**: إذن تسليم مُكتمل

```sql
-- القيد 1: إثبات الإيراد
-- ═══════════════════════════════════════

مدين: 1200 - العملاء .................... 15,000 دولار
      دائن: 4100 - إيرادات مبيعات الأقمشة ... 13,043 دولار
      دائن: 2300 - ضريبة القيمة المضافة .... 1,957 دولار

-- القيد 2: تسجيل التكلفة
-- ═══════════════════════════════════════

مدين: 5101 - تكلفة الأقمشة المباعة ....... 8,000 دولار
      دائن: 1403 - مخزون الأقمشة ............ 8,000 دولار

-- الوصف: بيع 40 متر قماش للعميل ABC
-- المرجع: Delivery Note #DN-2026-0045
```

**الجدول المتأثر**: `delivery_notes`, `roll_movements`

**نوع الحركة**: `sale`

### 🔄 3.3 مناقلة بين المستودعات

**الحدث**: نقل رولونات من مستودع لآخر

## 6. التنفيذ التقني {#6-التنفيذ}

### 🔧 6.1 Trigger للقيود التلقائية

```sql
-- Trigger لإنشاء قيد محاسبي عند حركة مخزون
CREATE OR REPLACE FUNCTION create_inventory_journal_entry()
RETURNS TRIGGER AS $$
DECLARE
    v_debit_account_id UUID;
    v_credit_account_id UUID;
    v_amount DECIMAL(15,2);
    v_entry_id UUID;
BEGIN
    -- حساب القيمة
    v_amount := NEW.quantity * COALESCE(NEW.unit_cost, 0);
    
    -- تحديد الحسابات بناءً على نوع الحركة
    CASE NEW.movement_type
        WHEN 'receive' THEN
            v_debit_account_id := get_account_id('inventory');
            v_credit_account_id := get_account_id('suppliers');
        WHEN 'sale' THEN
            v_debit_account_id := get_account_id('cogs');
            v_credit_account_id := get_account_id('inventory');
        WHEN 'adjustment_shortage' THEN
            v_debit_account_id := get_account_id('shortage_loss');
            v_credit_account_id := get_account_id('inventory');
        WHEN 'adjustment_surplus' THEN
            v_debit_account_id := get_account_id('inventory');
            v_credit_account_id := get_account_id('surplus_income');
        ELSE
            RETURN NEW; -- لا قيد للمناقلات
    END CASE;
    
    -- إنشاء القيد المحاسبي
    INSERT INTO journal_entries (
        tenant_id, company_id, entry_date, reference_type, reference_id, 
        description, status
    ) VALUES (
        NEW.tenant_id, NEW.company_id, NEW.movement_date, 
        'inventory_movement', NEW.id,
        'حركة مخزون: ' || NEW.movement_type,
        'posted'
    ) RETURNING id INTO v_entry_id;
    
    -- إضافة سطور القيد
    INSERT INTO journal_entry_lines (entry_id, account_id, debit, credit)
    VALUES 
        (v_entry_id, v_debit_account_id, v_amount, 0),
        (v_entry_id, v_credit_account_id, 0, v_amount);
    
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- تفعيل الـ Trigger
CREATE TRIGGER trg_inventory_accounting
    AFTER INSERT ON inventory_movements
    FOR EACH ROW
    EXECUTE FUNCTION create_inventory_journal_entry();
```

### 🔧 6.2 Function لحساب تكلفة المخزون

```sql
-- دالة حساب التكلفة بطريقة FIFO
CREATE OR REPLACE FUNCTION calculate_fifo_cost(
    p_material_id UUID,
    p_quantity DECIMAL
)
RETURNS DECIMAL AS $$
DECLARE
    v_total_cost DECIMAL := 0;
    v_remaining_qty DECIMAL := p_quantity;
    v_batch RECORD;
BEGIN
    -- جلب الدفعات بترتيب الاستلام (الأقدم أولاً)
    FOR v_batch IN
        SELECT id, current_quantity, unit_cost
        FROM inventory_batches
        WHERE product_id = p_material_id
        AND current_quantity > 0
        ORDER BY received_date ASC, created_at ASC
    LOOP
        IF v_remaining_qty <= 0 THEN
            EXIT;
        END IF;
        
        IF v_batch.current_quantity >= v_remaining_qty THEN
            v_total_cost := v_total_cost + (v_remaining_qty * v_batch.unit_cost);
            v_remaining_qty := 0;
        ELSE
            v_total_cost := v_total_cost + (v_batch.current_quantity * v_batch.unit_cost);
            v_remaining_qty := v_remaining_qty - v_batch.current_quantity;
        END IF;
    END LOOP;
    
    RETURN v_total_cost;
END;
$$ LANGUAGE plpgsql;
```

### 🔧 6.3 View لملخص المخزون المحاسبي

```sql
-- View لعرض قيمة المخزون لكل تصنيف
CREATE OR REPLACE VIEW v_inventory_valuation AS
SELECT 
    fm.group_id,
    fg.name_ar AS group_name,
    COUNT(DISTINCT fm.id) AS materials_count,
    SUM(fr.current_length) AS total_quantity,
    SUM(fr.current_length * fr.cost_per_meter) AS total_value,
    fm.unit AS unit
FROM fabric_materials fm
LEFT JOIN fabric_groups fg ON fm.group_id = fg.id
LEFT JOIN fabric_rolls fr ON fr.material_id = fm.id AND fr.status = 'available'
WHERE fm.status = 'active'
GROUP BY fm.group_id, fg.name_ar, fm.unit
ORDER BY total_value DESC;
```

## ✅ ملخص للمطور

### المطلوب تنفيذه:

1. **إنشاء الحسابات في شجرة الحسابات**:
   - حسابات المخزون (1400-1409)
   - حسابات تكلفة البضاعة المباعة (5100-5109)
   - حسابات الخسائر والإيرادات الأخرى

2. **ربط التصنيفات بالحسابات**:
   - كل `fabric_group` يرتبط بـ:
     - `default_inventory_account_id`
     - `default_cogs_account_id`

3. **تنفيذ Triggers**:
   - `trg_inventory_accounting` لإنشاء القيود التلقائية

4. **اختبار السيناريوهات**:
   - اختبار الاستلام → التحقق من زيادة المخزون
   - اختبار البيع → التحقق من تسجيل التكلفة
   - اختبار الجرد → التحقق من التسويات
