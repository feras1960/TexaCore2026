## 1. الوضع الحالي

### ✅ ما هو موجود:

#### 📝 جدول `audit_logs` (موجود ومُفعّل)
```sql
CREATE TABLE audit_logs (
    id UUID PRIMARY KEY,
    tenant_id UUID,           -- عزل بيانات المستأجر
    user_id UUID,             -- من قام بالعملية
    action VARCHAR(50),       -- create, update, delete, login, logout
    entity_type VARCHAR(100), -- اسم الجدول
    entity_id UUID,           -- ID السجل
    entity_name VARCHAR(255), -- اسم السجل للقراءة
    old_values JSONB,         -- القيم القديمة
    new_values JSONB,         -- القيم الجديدة
    changes JSONB,            -- الحقول المُغيّرة فقط
    ip_address INET,          -- عنوان IP
    user_agent TEXT,          -- المتصفح
    session_id UUID,          -- جلسة المستخدم
    metadata JSONB,           -- بيانات إضافية
    severity VARCHAR(20),     -- info, warning, error, critical
    created_at TIMESTAMPTZ    -- وقت الحدث
);
```

## 2. أفضل الممارسات العالمية

### 📚 معايير المحاسبة الدولية (IFRS/GAAP)

#### قاعدة أساسية:
> **"لا يجوز تعديل أو حذف أي قيد محاسبي مُرحل. التصحيح يتم بقيد عكسي."**

#### السبب:
1. **النزاهة المالية**: القوائم المالية يجب أن تعكس جميع العمليات
2. **المتطلبات القانونية**: مطابقة للتشريعات الضريبية
3. **التدقيق**: المراجعون يحتاجون رؤية التاريخ الكامل
4. **المساءلة**: من قام بماذا ومتى

### 🏢 ممارسات الأنظمة العالمية

| النظام | سياسة القيود المُرحلة | ملاحظات |
|--------|----------------------|---------|
| **SAP** | ❌ لا تعديل - قيد عكسي | صارم جداً |
| **Oracle Financials** | ❌ لا تعديل - قيد عكسي | مع صلاحيات استثنائية |
| **Microsoft Dynamics** | ⚠️ إلغاء ترحيل ثم تعديل | مع audit trail |
| **QuickBooks** | ⚠️ تعديل مع تحذير | للشركات الصغيرة |
| **Odoo** | ⚠️ إلغاء ترحيل ثم تعديل | مرن |

## 3. تحليل: التعديل المباشر vs إلغاء الترحيل

### 🔴 الخيار 1: التعديل المباشر للقيود المُرحلة

#### المزايا:
- ✅ سرعة في الإجراء
- ✅ مناسب للأخطاء البسيطة

#### العيوب:
- ❌ **مخالف للمعايير المحاسبية**
- ❌ يُفقد تاريخ التغييرات
- ❌ مشاكل مع التدقيق الخارجي
- ❌ مشاكل ضريبية محتملة
- ❌ صعوبة تتبع الفروقات

#### متى يُستخدم:
- ⚠️ فقط لتصحيح الأخطاء الإملائية (الوصف فقط)
- ⚠️ يتطلب صلاحية Super Admin

---

### 🟢 الخيار 2: إلغاء الترحيل → التعديل → إعادة الترحيل (موصى به)

#### المزايا:
- ✅ **مطابق للمعايير المحاسبية**
- ✅ سجل تدقيق كامل
- ✅ شفافية مع المراجعين
- ✅ تتبع من قام بالتغيير

#### العيوب:
- ⚠️ خطوات إضافية
- ⚠️ قد يؤثر على التقارير المؤقتة

#### متى يُستخدم:
- ✅ لأي تعديل على المبالغ
- ✅ لتغيير الحسابات
- ✅ لتغيير التاريخ

---

### 🟣 الخيار 3: القيد العكسي (الأفضل للتغييرات الجوهرية)

#### المزايا:
- ✅ **أفضل الممارسات المحاسبية**
- ✅ لا تعديل على أي سجل تاريخي
- ✅ سجل كامل للعمليات
- ✅ مقبول من جميع المراجعين

#### العيوب:
- ⚠️ يزيد عدد القيود
- ⚠️ قد يُربك المستخدمين الجدد

#### متى يُستخدم:
- ✅ بعد إغلاق الفترة المحاسبية
- ✅ للتعديلات الجوهرية
- ✅ عند طلب المراجع الخارجي

## 4. التوصيات

### 📋 مصفوفة الصلاحيات الموصى بها

| نوع المستند | الحالة | المستخدم العادي | المحاسب | المدير المالي | Super Admin |
|-------------|--------|-----------------|---------|---------------|-------------|
| **قيد محاسبي** | Draft | ✅ تعديل كامل | ✅ تعديل كامل | ✅ تعديل كامل | ✅ تعديل كامل |
| **قيد محاسبي** | Posted | ❌ | ⚠️ إلغاء ترحيل | ✅ إلغاء ترحيل | ✅ تعديل مباشر* |
| **قيد محاسبي** | Closed Period | ❌ | ❌ | ⚠️ قيد عكسي | ✅ قيد عكسي |
| **فاتورة** | Draft | ✅ | ✅ | ✅ | ✅ |
| **فاتورة** | Posted | ❌ | ⚠️ إلغاء | ✅ إلغاء | ✅ |
| **فاتورة** | Paid | ❌ | ❌ | ❌ | ⚠️ استثنائي |
| **حساب** | Active | ✅ الوصف فقط | ✅ تعديل | ✅ تعديل | ✅ تعديل |
| **حساب** | Has Transactions | ❌ حذف | ❌ حذف | ❌ حذف | ⚠️ أرشفة فقط |

> *التعديل المباشر يُسجّل في Audit Log مع تنبيه

---

### 🔒 قواعد الأمان الموصى بها

```
1. أي تعديل على مستند مُرحل يُسجّل في audit_logs
2. إلغاء الترحيل يتطلب سبب مكتوب
3. لا يمكن حذف أي مستند مُرحل (أرشفة فقط)
4. التعديل المباشر يتطلب صلاحية خاصة + تنبيه
5. كل تغيير يُرسل إشعار للمدير المالي
```

## 5. خطة التنفيذ

### المرحلة 1: تحسين Audit Trail (أولوية عالية)

#### 1.1 إضافة جدول تتبع الجلسات
```sql
CREATE TABLE user_sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    user_id UUID NOT NULL,
    session_token UUID,
    ip_address INET,
    user_agent TEXT,
    device_info JSONB,
    location JSONB,       -- country, city
    login_at TIMESTAMPTZ DEFAULT NOW(),
    logout_at TIMESTAMPTZ,
    last_activity_at TIMESTAMPTZ,
    is_active BOOLEAN DEFAULT true,
    logout_reason VARCHAR(50) -- manual, timeout, force_logout
);
```

#### 1.2 إضافة جدول تتبع النشاطات
```sql
CREATE TABLE user_activities (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID REFERENCES user_sessions(id),
    tenant_id UUID NOT NULL,
    user_id UUID NOT NULL,
    activity_type VARCHAR(50), -- page_view, action, report, export
    page_path TEXT,
    action_name VARCHAR(100),
    entity_type VARCHAR(100),
    entity_id UUID,
    metadata JSONB,
    duration_ms INT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### المرحلة 2: منطق التعديل في الـ Frontend

#### 2.1 للقيود المحاسبية
```typescript
const handleEditJournalEntry = async (entry) => {
    if (entry.status === 'posted') {
        // Show confirmation dialog
        const result = await showDialog({
            title: 'تعديل قيد مُرحل',
            message: 'هذا القيد مُرحل. ماذا تريد أن تفعل؟',
            options: [
                { id: 'unpost', label: 'إلغاء الترحيل ثم التعديل', recommended: true },
                { id: 'reverse', label: 'إنشاء قيد عكسي' },
                { id: 'cancel', label: 'إلغاء' }
            ]
        });
        
        if (result === 'unpost') {
            await unpostEntry(entry.id, 'تعديل بيانات');
            setMode('edit');
        } else if (result === 'reverse') {
            createReversingEntry(entry);
        }
    } else {
        setMode('edit');
    }
};
```

#### 2.2 للحسابات
```typescript
const handleEditAccount = async (account) => {
    const hasTransactions = await checkAccountTransactions(account.id);
    
    if (hasTransactions) {
        // Limit editable fields
        setEditableFields(['name', 'name_ar', 'description', 'is_active']);
    } else {
        // All fields editable
        setEditableFields('all');
    }
    
    setMode('edit');
};
```
