### ✅ الفكرة المُقترحة (موافق عليها 100%)

```

│              آلية التعديل المُبسّطة للشركات الصغيرة والمتوسطة         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   المستخدم يضغط "تعديل"                                              │
│          ↓                                                           │

│   │ النظام تلقائياً (خلف الكواليس):                          │      │

│   │   4. المستخدم يضغط "حفظ"                                 │      │

│   └──────────────────────────────────────────────────────────┘      │
│          ↓                                                           │
│   ✅ تم التعديل بسهولة                                               │
│   ✅ Audit Log مُحتفظ بكل التغييرات                                  │
│   ✅ المستخدم لا يرى التعقيدات                                       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 💡 لماذا هذا الأسلوب ممتاز:

| السبب | التفسير |
|-------|---------|
| **بساطة للمستخدم** | لا يحتاج فهم "الترحيل" و"إلغاء الترحيل" |
| **سلامة البيانات** | النظام يتأكد من التوازن قبل الحفظ |
| **Audit Trail** | كل خطوة مُسجّلة (unpost → edit → re-post) |
| **قابلية التوسع** | يمكن إضافة قيود لاحقاً حسب الحاجة |
| **مناسب للـ SMB** | الشركات الصغيرة لا تحتاج تعقيدات SAP |

## 3. السيناريوهات والحلول

### 📋 مصفوفة القرار:

| حالة الفترة | حالة السنة | التعديل المباشر | قيد تسوية | التوصية |
|-------------|------------|-----------------|-----------|---------|
| مفتوحة | مفتوحة | ✅ مسموح | ❌ لا حاجة | **تعديل مباشر** |
| مُغلقة | مفتوحة | ⚠️ بتحذير | ✅ الأفضل | **قيد تسوية** |
| مُغلقة | مُغلقة | ❌ ممنوع | ✅ إلزامي | **قيد تسوية في السنة الحالية** |

### 🔧 المنطق المُقترح في الكود:

```typescript
async function handleEditEntry(entry: JournalEntry) {
    const period = await getAccountingPeriod(entry.period_id);
    const fiscalYear = await getFiscalYear(entry.fiscal_year_id);
    
    // الحالة 1: الفترة والسنة مفتوحتان
    if (period.status === 'open' && fiscalYear.status === 'open') {
        // تعديل مباشر مع unpost → edit → re-post
        return allowDirectEdit(entry);
    }
    
    // الحالة 2: الفترة مغلقة لكن السنة مفتوحة
    if (period.status === 'closed' && fiscalYear.status === 'open') {
        return showDialog({
            title: 'الفترة المحاسبية مُغلقة',
            message: 'هذه الفترة مُغلقة. ما الذي تريد فعله؟',
            options: [
                {
                    id: 'adjustment',
                    label: 'إنشاء قيد تسوية (مُوصى به)',
                    description: 'إنشاء قيد جديد بالفرق في الفترة الحالية',
                    recommended: true
                },
                {
                    id: 'reopen',
                    label: 'إعادة فتح الفترة وتعديل',
                    description: 'يتطلب صلاحية المدير المالي',
                    requiresPermission: 'reopen_period'
                }
            ]
        });
    }
    
    // الحالة 3: السنة المالية مُغلقة
    if (fiscalYear.status === 'closed') {
        return showDialog({
            title: 'السنة المالية مُغلقة',
            message: 'لا يمكن تعديل قيود سنة مالية مُغلقة.',
            options: [
                {
                    id: 'adjustment_current_year',
                    label: 'إنشاء قيد تسوية في السنة الحالية',
                    description: 'هذا هو الإجراء المحاسبي الصحيح'
                },
                {
                    id: 'cancel',
                    label: 'إلغاء'
                }
            ]
        });
    }
}
```

## 4. التوصية النهائية

### ✅ الآلية المُعتمدة:

```
┌─────────────────────────────────────────────────────────────────────┐
│                     آلية التعديل الذكية                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  📊 إعدادات الشركة (يُحددها المدير):                                 │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │ ☑️ السماح بالتعديل المباشر على القيود المُرحلة              │   │
│  │ ☑️ إعادة الترحيل تلقائياً بعد الحفظ                         │   │
│  │ ☐ طلب موافقة للتعديل على القيود المُرحلة                    │   │
│  │ ☑️ تسجيل جميع التعديلات في سجل المراجعة                     │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  📋 قواعد ثابتة (لا يمكن تغييرها):                                   │
│  ├── ❌ لا تعديل على سنة مالية مُغلقة                                │
│  ├── ⚠️ تحذير عند التعديل على فترة مُغلقة                           │
│  └── ✅ كل تغيير يُسجّل في Audit Log                                 │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 📊 ملخص السلوك:

| السيناريو | السلوك | التأثير على الأرصدة |
|-----------|--------|---------------------|
| تعديل في فترة **مفتوحة** | ✅ تعديل مباشر | ✅ تُحدّث تلقائياً |
| تعديل في فترة **مُغلقة** (سنة مفتوحة) | ⚠️ تحذير + خيارات | ⚠️ تُحدّث إذا سمح |
| تعديل في سنة **مُغلقة** | ❌ ممنوع | ❌ قيد تسوية فقط |

### 🔑 النقطة الحاسمة:

> **التعديل داخل فترة/سنة مفتوحة لا يؤثر على الأرصدة الختامية أو الافتتاحية لأن هذه الأرصدة لم تُحسب بعد!**

> **التعديل على سنة مُغلقة يُفسد الترابط لأن الأرصدة الافتتاحية للسنة التالية مبنية على أرقام السنة المُغلقة.**

## 5. خطة التنفيذ

### الخطوة 1: إضافة إعدادات الشركة
```sql
-- في جدول company_settings أو accounting_settings
ALTER TABLE companies ADD COLUMN IF NOT EXISTS 
    edit_settings JSONB DEFAULT '{
        "allow_direct_edit_posted": true,
        "auto_repost_after_save": true,
        "require_approval_for_posted_edit": false,
        "allow_edit_closed_period": false,
        "log_all_edits": true
    }';
```

### الخطوة 2: تحديث Frontend

```typescript
// في UnifiedAccountingSheet
const handleEdit = async () => {
    const entry = data;
    const settings = await getCompanyEditSettings();
    
    if (entry.is_posted && settings.allow_direct_edit_posted) {
        // Unpost quietly
        await unpostEntry(entry.id);
        setMode('edit');
        setAutoRepostOnSave(true);
    }
};

const handleSave = async () => {
    await saveEntry(data);
    
    if (autoRepostOnSave) {
        await postEntry(data.id);
    }
    
    toast.success('تم حفظ التغييرات');
};
```

### الخطوة 3: التحقق من الفترات

```typescript
const checkPeriodStatus = async (entryDate: string) => {
    const period = await getAccountingPeriodByDate(entryDate);
    const fiscalYear = await getFiscalYearByDate(entryDate);
    
    return {
        periodOpen: period?.status === 'open',
        yearOpen: fiscalYear?.status === 'open',
        canEdit: period?.status === 'open' && fiscalYear?.status === 'open',
        canEditWithWarning: period?.status === 'closed' && fiscalYear?.status === 'open',
        requiresAdjustment: fiscalYear?.status === 'closed'
    };
};
```

## 📊 ملخص الإجابة على أسئلتك

| السؤال | الإجابة |
|--------|---------|
| **هل التعديل يؤثر على القيد الختامي؟** | فقط إذا كانت السنة مفتوحة (وهذا طبيعي) |
| **هل يؤثر على الأرصدة الافتتاحية للسنة التالية؟** | ❌ لا، إذا كانت السنة مُغلقة نمنع التعديل |
| **هل يمكن التعديل داخل الفترة فقط؟** | ✅ نعم، وهذا هو السلوك الافتراضي |
| **هل الآلية المُبسّطة مناسبة للـ SMB؟** | ✅ نعم، ممتازة ومرنة |
