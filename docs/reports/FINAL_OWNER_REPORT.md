## 🔒 الأمان والخصوصية

### نظام RLS (Row-Level Security):

**ما هو؟**  
نظام حماية على مستوى قاعدة البيانات يضمن أن كل مستخدم يرى **فقط** بيانات شركته.

**كيف يعمل؟**
```
1. المستخدم يسجل دخول
2. النظام يُحدد tenant_id الخاص به
3. Database تُطبق Policies تلقائياً:
   → المستخدم يرى بيانات tenant الخاص به فقط
   → لا يمكنه رؤية/تعديل بيانات tenants أخرى
4. حتى لو تم اختراق Application Code
   → Database محمية ومعزولة
```

**الإحصائيات:**
- ✅ **176 RLS Policy** نشطة
- ✅ **44 جدول** محمي
- ✅ **عزل 100%** بين المشتركين

**مثال عملي:**
```
مشترك A لديه 100 عميل
مشترك B لديه 200 عميل

عندما مستخدم من مشترك A يفتح قائمة العملاء:
→ يرى 100 عميل فقط (تابعين له)
→ لا يرى أي عميل من مشترك B
→ حتى لو حاول بـ SQL Injection!
```

## 💰 النظام المحاسبي

### المزايا:

#### 1. **Double-Entry Bookkeeping**
```
كل عملية مالية = قيد محاسبي متوازن
المدين = الدائن (دائماً!)

مثال: بيع بضاعة بـ 1150 ريال
┌────────────┬────────┬────────┐
│ الحساب    │ مدين  │ دائن  │
├────────────┼────────┼────────┤
│ العملاء   │ 1150   │   -    │
│ المبيعات  │   -    │ 1000   │
│ الضريبة   │   -    │  150   │
├────────────┼────────┼────────┤
│ المجموع   │ 1150   │ 1150   │ ✅
└────────────┴────────┴────────┘
```

#### 2. **أتمتة كاملة**
```
المستخدم:
1. ينشئ فاتورة مبيعات
2. يضغط "ترحيل"

النظام تلقائياً:
✅ ينشئ قيد محاسبي (3 بنود)
✅ يخصم المخزون
✅ يُحدّث رصيد العميل
✅ يربط الفاتورة بالقيد
✅ يتحقق من التوازن

الوقت: 0.5 ثانية! ⚡
```

#### 3. **حماية من الأخطاء**
```
❌ لا يمكن حفظ قيد غير متوازن
```
