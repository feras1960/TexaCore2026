## 1. ملخص تنفيذي

### الهدف
تطوير **بطاقة المادة** (`Material View Sheet`) لتكون مرجعاً شاملاً لكل ما يتعلق بالمادة، مع إضافة **صفحة دفتر المخزون** كصفحة رئيسية في قسم المستودعات.

### النطاق
| المكون | الحالة | المطلوب |
|--------|--------|---------|
| بطاقة المادة (Sheet) | ✅ موجود - يحتاج تحسين | إصلاح Scroll + تحسين + تبويبات جديدة |
| دفتر المخزون (صفحة جديدة) | ❌ غير موجود | إنشاء صفحة تقارير مخزون متقدمة |
| ميزة MDI (التبويبات المتعددة) | ✅ موجود في الشيت | توظيفها لفتح الفواتير/الرولونات |

### 2.2 آلية Scroll الحالية

```
UnifiedAccountingSheet
  └─ SheetContent (h-full, flex-col)
      └─ div.flex-col.h-full
          ├─ Header (ثابت)
          ├─ MainDocumentTabs (MDI - ثابت)
          └─ div.flex-1.overflow-hidden ← Content Area
              └─ SheetTabs
                  ├─ TabsList (sticky top-0 z-10) ← ✅ ثابت فعلاً
                  └─ Content
                      ├─ Ledger → div.flex-1.overflow-hidden ← خاص
                      └─ Others → ScrollArea.flex-1 ← ✅ يعمل
```

**النتيجة:** الـ Scroll يعمل على مستوى `UnifiedAccountingSheet` ✅
لكن بعض التبويبات الداخلية (مثل `MaterialInventoryTab`) لها `overflow-y-auto` خاص يتعارض.

### 2.3 ميزة MDI (Multi-Document Interface)

```typescript
// موجود في LedgerTab - عند النقر على قيد:
onEntryOpen={(entry) => {
    const newDoc = {
        id: entry.id,
        type: 'journal',
        title: entry.entry_number,
        data: entry,
        isClosable: true,
    };
    setOpenDocs(prev => [...prev, newDoc]);
    setActiveDocId(newDocId);
}}
```

**✅ الميزة موجودة وجاهزة** - يمكن استخدامها لفتح:
- بطاقة رولون وحركاته
- فاتورة بيع/شراء
- تفاصيل حركة مخزون

### التبويبات النهائية لبطاقة المادة:

#### 1️⃣ نظرة عامة (`overview`) - تحسين
```
• بطاقات إحصائيات: الكمية الإجمالية | المتاح | محجوز | عدد الرولونات | سعر البيع | القيمة
• معلومات المادة: الاسم | الكود | الفئة | الوحدة | الحالة
• المواصفات: العرض | النوع | الجودة | الموسم
• المخزون حسب المستودع (جدول مصغر): المستودع | الكمية | عدد الرولونات | المتاح | الحالة

```

#### 2️⃣ المخزون والرولونات (`inventory`) - تحديث شامل
```
• بطاقات ملخص: إجمالي الأمتار | عدد الرولونات | المتاح | المحجوز | القيمة

  - المستودع | الموقع | الكمية | المحجوز | المتاح | عدد الرولونات | الحالة
  - مع صف إجمالي (Footer)
• جدول الرولونات (NexaDataTable):
  - رقم الرولون | اللون | الطول الأصلي | الحالي | المتاح | العرض | الجودة | المستودع | الحالة
  - عند النقر → فتح بطاقة الرولون في MDI Tab
```

```
• جدول الحركات (NexaDataTable):
  - التاريخ | النوع | المرجع | من | إلى | الكمية | القيمة | الرصيد | المستخدم
  - عند النقر على المرجع → فتح الفاتورة/القيد في MDI Tab
```

## 📌 الرأي الاحترافي

### لماذا هذا النهج الأفضل:

1. **فصل المسؤوليات**: بطاقة المادة = تفاصيل مادة واحدة | دفتر المخزون = نظرة شاملة على كل المواد
2. **إعادة استخدام المكونات**: كلاهما يستخدم NexaDataTable بنفس الأنماط
3. **MDI كميزة تنافسية**: القدرة على فتح فاتورة/رولون من داخل بطاقة المادة دون مغادرتها
4. **الفلاتر الديناميكية**: تتغير حسب فئة المادة (قماش ≠ أحذية ≠ إكسسوار)
5. **تلوين الصفوف**: ميزة المراجعة الموجودة في المشروع القديم - مفيدة جداً للمراجعة

### الفرق عن المشروع القديم:
| الجانب | المشروع القديم | المشروع الجديد (المقترح) |
|--------|---------------|------------------------|
| الجداول | Table عادي | NexaDataTable (بحث + فلاتر + تصدير + تخصيص أعمدة) |
| الترجمة | Inline | i18n keys |
