## 1️⃣ 📞 نظام الاتصالات — Grandstream + n8n

### 🏗️ البنية المعمارية

```
                    ┌─────────────────────┐
                    │  Grandstream UCM63xx │
                    │  (PBX / مقسم)        │
                    └───────┬─────────────┘
                            │ AMI/API
                            ▼
                    ┌─────────────────────┐
                    │      n8n Hub        │
                    │  (الوسيط المركزي)    │
                    └──┬──────┬──────┬────┘
                       │      │      │
              ┌────────┘      │      └────────┐
              ▼               ▼               ▼
        ┌──────────┐  ┌──────────────┐  ┌──────────┐
        │ Supabase │  │ Telegram Bot │  │ Gemini AI│
        │ (تسجيل) │  │ (إشعارات)   │  │ (تحليل) │
        └──────────┘  └──────────────┘  └──────────┘
```

### 🔧 متطلبات الأجهزة

| المكون | الوصف | الحالة |
|--------|-------|--------|
| **Grandstream UCM6304/6308** | مقسم IP-PBX يدعم AMI API | ⏳ مطلوب |
| **هواتف Grandstream GXP/GRP** | أجهزة VoIP للموظفين | ⏳ مطلوب |
| **n8n** | الوسيط بين المقسم والنظام | ✅ مثبت |

### 📌 الميزات المطلوبة بالتفصيل

#### 1.1 — IVR (الرد الآلي التفاعلي)
```
📞 العميل يتصل
    │
    ▼
🤖 IVR يرد: "مرحباً بكم في TexaCore"
    │
    ├── اضغط 1 ← المبيعات
    ├── اضغط 2 ← الدعم الفني
    ├── اضغط 3 ← الشكاوى
    ├── اضغط 4 ← حجز موعد
    └── اضغط 0 ← التحدث مع موظف
    │
    ▼
n8n يسجّل اختيار العميل + يحوّل المكالمة
```

**التنفيذ عبر n8n:**
- Grandstream يرسل webhook لـ n8n عند كل حدث مكالمة
- n8n يتحقق من رقم المتصل في قاعدة البيانات (customers)
- n8n يقرر التحويل بناءً على: المانيجر المفضل → آخر مانيجر تحدث معه → أي مانيجر متاح

#### 1.2 — تسجيل المكالمات وتحليلها بالـ AI
```
📞 المكالمة تنتهي
    │
    ▼
Grandstream يحفظ التسجيل (WAV/MP3)
    │
    ▼
n8n يأخذ التسجيل
    │
    ├──► Google Speech-to-Text (تحويل صوت → نص)
    │
    ├──► Gemini AI يحلل:
    │      • نوع المحادثة: دعم فني / شكوى / حجز / استفسار
    │      • الحالة النفسية: راضي 😊 / محايد 😐 / غاضب 😠
    │      • ملخص المكالمة (3-5 أسطر)
    │      • إجراء مطلوب (متابعة / لا شيء / تصعيد)
    │
    └──► Supabase (يحفظ كل شيء في جدول call_logs)
```

#### 1.3 — الاتصال السريع (Click-to-Call)
```
المانيجر في TexaCore ERP
    │
    ▼
يضغط 📞 بجانب اسم العميل
    │
    ▼
Frontend يرسل طلب لـ n8n webhook:
  { action: "call", customer_phone: "+905...", manager_ext: "101" }
    │
    ▼
n8n ──API──► Grandstream UCM:
  1. يتصل بخط المانيجر (ext 101)
  2. المانيجر يرفع السماعة
  3. المقسم يتصل بالعميل تلقائياً
  4. المكالمة تبدأ
```

#### 1.4 — لوحة المكالمات الحية (Real-time)
```
Dashboard في TexaCore ERP
    │
    ▼
n8n يسحب حالة المكالمات كل 5 ثواني من Grandstream API
    │
    ▼
يعرض في الواجهة:
  ┌──────────────────────────────────────────┐
  │ 📞 المكالمات الجارية (3)                  │
  ├──────────────────────────────────────────┤
  │ 🟢 أحمد ← العميل محمد   | 00:03:45     │
  │ 🟢 خالد ← العميل فاطمة  | 00:01:20     │
  │ 🟡 سارة ← في الانتظار    | 00:00:30     │
  └──────────────────────────────────────────┘
```

### 📊 الجداول المطلوبة لنظام الاتصالات

| الجدول | الوصف | يرتبط بـ |
|--------|-------|---------|
| `call_logs` | سجل جميع المكالمات | customers, user_profiles |

### 📌 سيناريو مفصّل

#### السيناريو أ: عميل جديد يدخل المتجر
```
1. يرى عينة قماش معجب بها
2. يمسح QR Code بجواله
3. يفتح Telegram Bot (@Texacorebot)
4. البوت يسأله: "مرحباً! أدخل رقم هاتفك 📱"
5. يدخل رقمه → البوت يبحث في قاعدة البيانات
6. لم يجده → يعرض:
   📋 "قماش تركي - بوليستر 100%"
   💰 "السعر: 45 SAR/متر" (سعر التجزئة)
   📦 "متوفر بالألوان: أحمر، أزرق، أسود"
   🛒 "للطلب تواصل مع أقرب مندوب"
   📝 "سجّل بياناتك للحصول على أسعار خاصة"
```

#### السيناريو ب: عميل مسجّل (VIP)
```
1. يمسح QR Code
2. البوت يتعرف عليه من رقم Telegram
3. يعرض:
   📋 "قماش تركي - بوليستر 100%"
   💰 "سعرك الخاص: 38 SAR/متر" (سعر الجملة)
   📦 "آخر طلبية لك: 500 متر بتاريخ 15/01/2026"
   🎁 "عرض خاص: خصم 5% للطلبات فوق 1000 متر"
   📞 "مانيجرك: أحمد (اضغط للاتصال)"
```

#### السيناريو ج: مانيجر يمسح QR
```
1. يمسح QR Code
2. البوت يتعرف عليه كموظف
3. يعرض بيانات إدارية كاملة:
   📋 "قماش تركي - بوليستر 100% | كود: FM-2024-0045"
   📦 "المخزون الحالي:"
   ┌─────────────┬────────┬──────────┐
   │ المستودع    │ رولونات│ أمتار    │
   ├─────────────┼────────┼──────────┤
   │ المستودع 1  │ 12     │ 1,440 م  │
   │ المستودع 3  │ 5      │ 600 م    │
   │ قيد الشحن   │ 20     │ 2,400 م  │
   └─────────────┴────────┴──────────┘
   💰 "سعر التكلفة: 28 SAR | التجزئة: 45 SAR | الجملة: 38 SAR"
   📊 "حركة الشهر: بيع 800م | وارد 2400م"
   📍 "الموقع: رف A3, صف 7"
```

## ⚠️ ملاحظات مهمة

### 1. Grandstream UCM API
- يحتاج جهاز **Grandstream UCM6304** أو أعلى (يدعم API)
- الاتصال عبر **AMI (Asterisk Manager Interface)** أو **HTTP API**
- n8n يتصل بالمقسم عبر الشبكة المحلية (LAN)
- **لا يعمل عن بعد** إلا مع VPN أو Port Forwarding

### 2. QR Code + Telegram
- لا يحتاج أي أجهزة إضافية
- يعمل فوراً مع البوت الموجود `@Texacorebot`
- يحتاج فقط: طباعة QR Codes ووضعها على العينات

### 3. الأولوية المقترحة
```
المرحلة 1 (فوري):     📱 QR Code الذكي (أسهل وأسرع تأثير)
المرحلة 2 (أسبوع):    🔔 إشعارات الأدوار
المرحلة 3 (أسبوعين):  📞 نظام الاتصالات (يحتاج أجهزة)
المرحلة 4 (شهر):      📍 نظام الشحن الكامل
```
