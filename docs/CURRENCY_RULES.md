# 🎯 قواعد إلزامية: القيود المحاسبية متعددة العملات

## المبدأ الأساسي — اقرأ بعناية!

عند التعامل مع القيود المحاسبية (journal entries)، يوجد **نوعان من المبالغ لكل سطر**:

### 1. المبلغ بالعملة الأصلية (Foreign Currency - FC)
- هذا هو المبلغ الذي يُدخله المستخدم
- يُخزن في حقول: `debit_fc` و `credit_fc`
- يظهر للمستخدم في أعمدة "مدين" و "دائن"
- **لا يُضرب بسعر الصرف أبداً!**

### 2. المبلغ بالعملة المحلية (Local Currency)
- هذا هو المبلغ المحسوب تلقائياً
- يُخزن في حقول: `debit` و `credit` (= debit_local / credit_local)
- يُحسب بالمعادلة: `المبلغ الأصلي × سعر الصرف`
- يُستخدم من `post_journal_entry` RPC لتحديث أرصدة الحسابات
- يُستخدم للتقارير المالية

---

## ❌ خطأ شائع — لا تفعل هذا!
```
المستخدم يدخل: 14,000 USD
سعر الصرف: 43.89

❌ خطأ: عرض 614,460 في عمود "مدين"
```

## ✅ الطريقة الصحيحة
```
المستخدم يدخل: 14,000 USD
سعر الصرف: 43.89

✅ صحيح:
   - عمود "مدين": 14,000 (بالعملة الأصلية USD)
   - عمود "العملة": USD
   - حقل debit (محلي): 614,460 (محسوب للـ RPC والتقارير)
```

## هيكل البيانات في قاعدة البيانات (journal_entry_lines)

```typescript
interface JournalEntryLine {
  // ═══════════════════════════════════════════════════════
  // ما يُدخله المستخدم ويراه (العملة الأصلية)
  // ═══════════════════════════════════════════════════════
  currency: string;           // 'USD' | 'EUR' | 'UAH' | etc
  exchange_rate: number;      // سعر الصرف (مثل 43.89)
  
  debit_fc: number;           // المدين بالعملة الأصلية (14,000)
  credit_fc: number;          // الدائن بالعملة الأصلية
  
  // ═══════════════════════════════════════════════════════
  // ما يُحسب تلقائياً (العملة المحلية - لا يُعرض في الجدول)
  // ═══════════════════════════════════════════════════════
  debit: number;              // = debit_fc × exchange_rate (614,460)
  credit: number;             // = credit_fc × exchange_rate
}
```

## كيف يعمل post_journal_entry RPC

```sql
-- الـ RPC يقرأ debit/credit (المحلي) مباشرة:
UPDATE chart_of_accounts
SET current_balance = current_balance + (v_line.debit - v_line.credit)
WHERE id = v_line.account_id;
```

**لذلك:**
- `debit`/`credit` يجب أن تكون **بالعملة المحلية** (مضروبة × سعر الصرف)
- `chart_of_accounts.current_balance` دائماً **بالعملة المحلية**

## قواعد العرض في واجهة JournalVoucherTab

### جدول سطور القيد:

| # | الحساب | البيان | مدين | دائن | العملة | سعر الصرف |
|---|--------|--------|------|------|--------|-----------|
| 1 | المورد | رصيد افتتاحي | **14,000** | - | USD | 43.89 |

- عمود "مدين" = `debit_fc` (القيمة الأصلية بدون ضرب!)
- عمود "دائن" = `credit_fc` (القيمة الأصلية بدون ضرب!)

### بادج الرصيد بجانب الحساب:
- إذا عملة الحساب = محلية → يعرض `current_balance` مباشرة
- إذا عملة الحساب ≠ محلية → يعرض `current_balance ÷ exchange_rate` بعملة الحساب

## عند الحفظ (journalEntriesService.create)

```typescript
const saveLine = (userInput: { debit: number; credit: number; currency: string; exchange_rate: number }) => {
  return {
    // المبالغ الأصلية — كما أدخلها المستخدم
    debit_fc: userInput.debit,                                    // 14,000
    credit_fc: userInput.credit,                                  // 0
    
    // المبالغ المحلية — محسوبة (لـ RPC)
    debit: userInput.debit * userInput.exchange_rate,              // 614,460
    credit: userInput.credit * userInput.exchange_rate,            // 0
    
    currency: userInput.currency,                                  // 'USD'
    exchange_rate: userInput.exchange_rate,                         // 43.89
  };
};
```

## عند القراءة والعرض

```typescript
const displayLine = (dbLine: JournalEntryLine) => {
  const rate = dbLine.exchange_rate || 1;
  const hasFC = rate > 1;
  
  return {
    // ما يظهر في أعمدة مدين/دائن
    debit:  hasFC ? (dbLine.debit_fc > 0 ? dbLine.debit_fc : dbLine.debit / rate) : dbLine.debit,
    credit: hasFC ? (dbLine.credit_fc > 0 ? dbLine.credit_fc : dbLine.credit / rate) : dbLine.credit,
    currency: dbLine.currency,
    exchange_rate: rate,
  };
};
```

## أرصدة العملاء والموردين (Sub-Ledger)

```
┌─────────────────────────────────┐    ┌─────────────────────────────────┐
│   الدفتر العام (General Ledger)  │    │  دفتر المورد (Sub-Ledger)       │
│   العملة: المحلية (UAH) دائماً   │    │  العملة: عملة المورد (USD)       │
├─────────────────────────────────┤    ├─────────────────────────────────┤
│ حساب 2111-001 مصانع بورصة      │    │ مورد SUP-001 مصانع بورصة       │
│ رصيد: 614,460 ₴ (غريفن)        │    │ رصيد: 14,000 $ (دولار)          │
│                                 │    │                                 │
│ يُستخدم في: الميزانية، الأستاذ   │    │ يُستخدم في: كشف حساب المورد     │
│ مصدر البيانات: debit/credit     │    │ مصدر البيانات: debit_fc/credit_fc│
└─────────────────────────────────┘    └─────────────────────────────────┘
```

## ملخص سريع

| الحقل | المصدر | يُعرض في الجدول؟ | يُستخدم بواسطة RPC؟ |
|-------|--------|------------------|---------------------|
| `debit_fc` | إدخال المستخدم | ✅ نعم (عمود مدين) | ❌ لا |
| `credit_fc` | إدخال المستخدم | ✅ نعم (عمود دائن) | ❌ لا |
| `debit` | حساب تلقائي | ❌ لا | ✅ نعم (يُحدّث current_balance) |
| `credit` | حساب تلقائي | ❌ لا | ✅ نعم (يُحدّث current_balance) |
